B1. 생물학 배경 · 문제 정의

"왜 genomic AI가 필요한가" 동기 라인. 출구 = C4(패러다임).

C1 정밀의료 (Precision Medicine) ⭐低선행: 없음 ▶
🎯 직관"같은 병명이라도 사람마다 유전적 원인이 다르니, 유전체를 읽어 개인 맞춤 치료를 하자."
📖 정의개인의 유전적 구성·환경·생활습관·임상데이터에 기반해 예방·진단·치료를 맞춤화하는 의료 접근.
🌍 실세계 시나리오
🌍 유방암 환자 A·B·C가 같은 "유방암" 진단을 받아도, A는 HER2 증폭, B는 BRCA1 변이, C는 PIK3CA 변이를 가진다. → A는 anti-HER2, B는 PARP 억제제, C는 PI3K 억제제로 서로 다른 표적치료. 이 "변이→치료 매칭"을 자동화하려면 유전체를 빠르고 정확히 해석하는 AI가 필요하다 — 이것이 본 과목의 출발점.
🔍 작동 원리 (4단계)① Same Disease → ② Genetic Testing(DNA/RNA seq, CNV, MSI/TMB) → ③ Different Alterations → ④ Different Targeted Treatments.
⚠️ 함정
"정밀의료=비싼 검사"가 아니라, 변이별 표적 매칭이 핵심. 병명이 아니라 변이가 치료를 정한다.
📝 시험 포인트"왜 genomic AI가 필요한가"의 동기 설명용. 깊은 출제보다 서론.출처: Notion Day 2
🔗 연결응용 → C2(변이 종류), C3(탐지 파이프라인)
C2 유전 변이의 종류 (Genetic Variations) ⭐低선행: 없음(생물 배경) ▶
🎯 직관변이는 "한 글자 수준"부터 "염색체 통째" 수준까지 스케일이 다양하다.
📖 정의/분류
  • Small-scale (DNA 서열 수준): SNV(점돌연변이, 예 겸상적혈구 HBB A→T), Indel(작은 삽입/결실, 예 낭포성섬유증 ΔF508 3bp 결실→frameshift), 짧은 반복 확장(헌팅턴 CAG repeat, 정상 10–35 → 질병 ≥36), 유전자 수준 CNV.
  • Structural (염색체 수준): 결실(5p→Cri-du-chat), 중복(PMP22→Charcot-Marie-Tooth 1A), 역위(Factor VIII→혈우병 A), 전좌(t(9;22)→필라델피아 염색체→CML), 대규모 CNV(22q11.2 deletion→DiGeorge).
🌍 실세계 시나리오
🌍 겸상적혈구빈혈은 HBB 유전자의 단 한 글자(A→T) 변화로 발생한다. 이 "한 글자가 질병을 좌우"하는 사실이 → 왜 gLM이 single-nucleotide 해상도(C5)를 중시하고, HyenaDNA가 character-level을 고집하는지의 생물학적 근거다.
📊 표 인사이트
변이 스케일이 클수록(SNV→전좌) 탐지 모델이 봐야 할 문맥 길이가 길어진다 → long-range 아키텍처(C15)의 동기와 연결.
⚠️ 함정
SNV(개체의 변이)와 SNP(집단 다형성)를 혼동하기 쉬움. 시험은 변이 유형↔질병 예시 매칭형.
📝 시험 포인트변이 유형↔질병 예시 매칭. "SNP/SNV가 단백질 기능·질병을 바꾼다"가 single-nt 중시 이유(C8, C20과 연결).출처: Notion Day 2
🔗 연결응용 → C5(single-nt), C8(CNN pileup), C20(바이러스 분류)
C3 변이 탐지 AI 파이프라인 (전통 ML vs gLM) ⭐中선행: C2 ▶
🎯 직관"변이를 찾는 두 길 — (A) 손으로 만든 feature + 트리모델, (B) raw 서열을 통째로 먹는 gLM."
📖 정의 (공통 파이프라인)환자 DNA → 시퀀싱(raw reads, FASTQ) → 레퍼런스 정렬·변이 호출(VCF) → 변이 주석(gene / population freq gnomAD / functional SnpEff·VEP / disease DB COSMIC·ClinVar·HGMD) → AI 예측(pathogenic vs benign).
🌍 실세계 시나리오
🌍 임상 유전학자가 환자 VCF에서 발견한 미지의 변이가 "병을 일으키는가(pathogenic) 아닌가(benign)"를 판정해야 한다. (A) 경로는 conservation 점수·SIFT/PolyPhen 등 사람이 설계한 feature를 XGBoost에 넣는다. (B) 경로는 그 변이 주변 raw DNA를 사전학습된 foundation model에 직접 넣어 0.91 같은 pathogenic 확률을 얻는다 — feature 설계 노동이 사라진다.

⚠️ 함정
(B)가 항상 우월한 게 아니라, feature를 사람이 설계하느냐 모델이 학습하느냐가 본질 차이. (추출본 p.36: 범용 foundation model이 특화 태스크엔 못 미칠 수 있음.)

C4 서열을 언어로 — gLM의 기본 패러다임 ⭐高관문선행: C3 ▶
🎯 직관"DNA를 문장, 뉴클레오타이드를 글자로 보고 NLP의 pretrain → finetune을 그대로."
📖 정의gLM은 ① intricate genomic grammar와 distant regulatory/functional interaction을 식별하고, ② 아키텍처·토크나이징·사전학습 전략·평가 방법론을 다운스트림 태스크에 맞게 최적화하는 모델군.
🌍 실세계 시나리오
🌍 영어 LLM이 위키피디아 전체로 사전학습 후 감정분류로 파인튜닝되듯, gLM은 인간 유전체 30억 bp(라벨 없음)로 사전학습한 뒤 "이 부위가 enhancer인가" 같은 라벨 태스크로 파인튜닝된다. 생물학자는 라벨 데이터가 적어도(수천 개) 강력한 표현을 재사용할 수 있다.
🔍 작동 원리 (학습 2단계)Random gLM → (대량 unlabeled genomic data) Pretraining → Pretrained gLM → (task data) Fine-tuning → Task-adapted gLM. 데이터 원천: human / multi-species / regulatory / prokaryotic·viral genome.

이후 모든 모델(DNABERT, NT, HyenaDNA, ProtBERT)은 이 2단계 틀의 변주 — "어떤 토큰 × 어떤 아키텍처 × MLM/CLM"의 조합 차이일 뿐.

서열을 어떻게 토큰·벡터로 바꾸는가. 세 방식(C5·C6·C7)의 trade-off 비교가 핵심 — 함께 학습.

C5 토크나이징 ① Single-nucleotide embedding ⭐高선행: C4 ▶
🎯 직관"글자(A/C/G/T) 하나하나를 토큰으로 — SNP 효과를 직접 모델링하지만 시퀀스가 길어진다."
📖 정의각 뉴클레오타이드를 vocabulary 토큰으로 보고 학습가능 dense vector로 임베딩. → base-level 정보 보존, SNP 효과 직접 모델링.
📖 대비 (One-hot encoding)각 염기를 distinct binary vector로(A=[1,0,0,0], C=[0,1,0,0]…). → bias·정보손실 없음, but 입력 시퀀스가 매우 길어짐(학습 dense embedding이 아님).
🌍 실세계 시나리오
🌍 겸상적혈구의 A→T 변이(C2)를 모델이 포착하려면, 그 한 글자가 곧 한 토큰의 변화여야 한다. single-nt 토큰화는 이 1:1 대응 덕에 변이 효과 예측(Nucleotide GPT, HyenaDNA)에서 강하다.
✏️ 단계별 풀이 예제 — One-hot 행렬 만들기 (seq = "ACTACGACT")
1단계
vocabulary 고정 → 행 순서 [A, C, G, T]
2단계
각 위치를 열로. 위치1 'A' → [1,0,0,0]ᵀ
3단계
위치2 'C' → [0,1,0,0]ᵀ, 위치3 'T' → [0,0,0,1]ᵀ …
4단계
9개 위치를 이어 붙이면 4×9 행렬 완성
5단계 · 관찰
길이 9 서열이 4×9=36 수. 길이 L이면 4×L → L이 커지면 입력 폭발 → single-nt의 약점(긴 시퀀스).
📊 표 인사이트
Nucleotide GPT Table 3에서 single-nucleotide가 대체로 최고 성능(C14). 단 일부 태스크는 6-mer가 우세 → 태스크 의존.
⚠️ 함정
single-nt(학습 임베딩) ≠ one-hot(고정 binary). 둘 다 base-level이지만 전자는 dense·학습가능, 후자는 sparse·고정.
📝 시험 포인트 ⭐single-nt vs k-mer vs BPE trade-off 비교(C6·C7과 묶음)가 핵심. 서술형 대비: "해상도(SNP 직접) vs 시퀀스 길이"의 교환관계를 설명.출처: Notion Day 2

C6 토크나이징 ② k-mer tokenization ⭐高선행: C5 ▶
🎯 직관"연속한 k개 염기를 한 단어로 묶기 — non-overlapping은 꼬리가 잘리고, overlapping은 중복이 심하다."
📖 정의연속된 k개 뉴클레오타이드를 하나의 토큰으로.
  • Non-overlapping: 겹치지 않게 자름 → trailing base(꼬리) 손실 가능. (3-mer: ACT|ACG|ACT)
  • Overlapping: 한 칸씩 밀며 자름 → highly redundant. (6-mer: ACTACG / CTACGA / TACGAC / ACGACT)
🌍 실세계 시나리오
🌍 DNABERT(C13)는 3~6-mer 토큰을 쓴다. "단어"가 생기면 코돈(3염기=1아미노산) 같은 생물학적 단위와 정렬되기 쉽지만, overlapping을 쓰면 인접 토큰이 5/6 글자를 공유해 중복 정보가 모델을 비효율적으로 만든다.
✏️ 단계별 풀이 예제 — "ACTACGACT"를 3-mer로
1단계 · non-overlapping
3칸씩 끊기: pos1-3 ACT, pos4-6 ACG, pos7-9 ACT → [ACT, ACG, ACT], 토큰 3개, 꼬리 손실 없음(길이 9가 3의 배수라서).
2단계
만약 길이 10(…ACTA)이면 마지막 'A' 1글자가 3-mer를 못 채워 버려짐 → trailing base 손실.
3단계 · overlapping 6-mer
한 칸씩: ACTACG, CTACGA, TACGAC, ACGACT → 토큰 4개.
4단계 · 관찰
ACTACG와 CTACGA는 5글자 공유 → highly redundant. 토큰 수 ≈ L−k+1로 길이도 거의 안 줄음.
답
non-overlap=정보 손실(꼬리)/짧음, overlap=중복/긺.
📊 표 인사이트
HyenaDNA·NT 비교표에 k-mer 버전 등장(C16, C14). DNABERT는 k-mer 채택.
⚠️ 함정
"overlapping이 정보가 더 많다"는 오해 — 실제론 중복이라 모델 부담만 늘 수 있다.
📝 시험 포인트 ⭐"왜 overlapping이 중복인가 / non-overlapping은 무엇을 잃는가" 설명. 서술형 대비: 같은 서열에 두 방식을 적용해 토큰 수·손실·중복을 대조.출처: Notion Day 2

C7 토크나이징 ③ Byte Pair Encoding (BPE) ⭐中선행: C5, C6 ▶
🎯 직관"자주 같이 나오는 글자쌍을 반복적으로 합쳐 새 토큰을 만든다 — 데이터가 vocabulary를 정한다."
📖 정의/메커니즘base vocabulary {A,C,T,G}에서 시작 → 가장 빈번한 인접 토큰 쌍을 새 토큰으로 반복 병합 → 목표 vocabulary 크기 도달까지.
🌍 실세계 시나리오
🌍 DNABERT-2(현대 gLM 추세)가 BPE를 채택. genome마다 자주 등장하는 motif(예 프로모터의 TATA box)가 자동으로 하나의 토큰이 된다 — 데이터가 "의미 단위"를 스스로 발견. Notion: "consecutive하게 의미 있는 단위로 길이를 넓혀가는 과정."
✏️ 단계별 풀이 예제 — BPE 병합 추적 (seq에 ACTAC… 반복 가정)
iter0
vocab = {A, C, T, G}. seq를 글자 단위로 토큰화.
iter1
가장 빈번한 인접쌍이 (A,C)라 가정 → 병합 → vocab = {A,C,T,G, AC}. seq에서 모든 AC가 한 토큰으로.
iter2
이제 가장 빈번한 인접쌍이 (AC, T) → 병합 → vocab += ACT.
iter3
(ACT, AC) 빈번 → 병합 → vocab += ACTAC.
답
반복마다 vocab이 확장되고 토큰 길이가 길어진다 — 빈도가 토큰 경계를 결정. 목표 크기 도달 시 종료.
📊 표 인사이트
반복마다 seq의 병합 위치와 vocabulary 확장을 추적하는 표 → 데이터 의존적 토큰화의 시각화.
⚠️ 함정
BPE는 빈도 기반이라 도메인(genome)마다 토큰이 달라짐 — 한 vocab을 다른 종에 그대로 쓰면 최적이 아닐 수 있음.
📝 시험 포인트 ⭐3가지 토크나이징(C5–C7) 장단 비교 = 가장 빈출 가능 modeling 질문. 서술형 대비: single-nt(고정·base) / k-mer(고정·창문) / BPE(데이터 학습)의 vocabulary 결정 방식과 장단을 한 표로.출처: Notion Day 2

C8 모델 아키텍처 ① CNN (변이 탐지) ⭐中선행: C3 ▶
🎯 직관"정렬된 read를 이미지(pileup)로 바꿔 CNN으로 보면 국소 패턴(motif)을 잘 잡지만, 먼 거리 의존성은 놓친다."
📖 정의/메커니즘 (pileup 이미지화)환자 N명 vs 건강인 M명 → 시퀀싱(WGS/Exome, FASTQ) → reads를 레퍼런스에 정렬(BWA-MEM) → 미스매치 패턴 → 매핑 결과를 이미지(픽셀, 위치×read depth, 예 221bp×depth100)로 변환 → 환자=라벨1/건강인=라벨0으로 CNN 학습(손실 Binary Cross Entropy, 역전파).
🌍 실세계 시나리오
🌍 DeepVariant/NeuSomatic 계열은 정렬된 read 더미(pileup)를 사진처럼 만들어, 변이 자리에 생기는 색·정렬 패턴을 CNN이 "이미지 분류"하듯 본다. 작은 필터가 지역을 슬라이딩하므로 local motif(변이 주변 패턴)에 강하다.
📊 표 인사이트 (CNN vs Transformer, p.20)
CNN = 작은 필터가 지역 슬라이딩(지역 합성곱)→지역 특징맵. Transformer = self-attention으로 모든 위치↔모든 위치.
✏️ 단계별 풀이 예제 — "왜 CNN은 long-range를 놓치나" 추론
1단계
필터 크기 k(예 3)인 1층 conv → 한 출력 뉴런은 입력 3칸만 본다(수용영역=3).
2단계
2층 쌓으면 수용영역≈5, L층이면 ≈2L+1 — 선형으로만 커짐.
3단계
1000bp 떨어진 두 위치를 연결하려면 수백 층 필요 → 비현실적.
4단계
반면 self-attention(C11)은 1층에서 모든 위치쌍을 직접 연결.
답
CNN의 수용영역이 층수에 선형이라 long-range가 약함 → Transformer/Hyena 동기.
⚠️ 함정
"CNN은 무조건 약하다"가 아니라, local motif엔 강하고 long-range엔 약하다는 trade-off.
📝 시험 포인트 ⭐"CNN은 왜 local에 강하고 long-range에 약한가" → attention과 대조(C19, Hyena 동기 C15). 서술형 대비: 수용영역이 층수에 선형이라는 점을 근거로 설명.출처: Notion Day 2 "CNN: good for motif detection"

C9 NeuSomatic (CNN으로 변이 탐지) ⭐中선행: C8 ▶
🎯 직관"후보 somatic 변이 주변을 매트릭스로 만들어 deep CNN으로 변이 유형·길이·위치를 분류."

🌍 실세계 시나리오
🌍 종양 조직은 정상세포와 섞여 변이 비율(allele freq)이 낮고 시퀀싱 에러와 구분이 어렵다. NeuSomatic은 tumor·normal을 나란히 매트릭스로 만들어 CNN이 "진짜 somatic 변이 vs 에러"를 구별 — 암 정밀의료(C1)의 핵심 도구.

C15 Hyena (Hyena Hierarchy) ⭐高난개념선행: C8, C19(attention quadratic) ▶
🎯 직관"Attention의 \(O(L^2)\) 비용을 피하면서 긴 서열을 다루는 subquadratic 대체 — 긴 합성곱 + 데이터 제어 게이팅."
📖 정의 (Def 3.1, Order-N Hyena Operator)입력 projection \((v, x^1,\dots,x^N)\)과 학습가능 필터 \(h^1,\dots,h^N\)에 대해 재귀
\[ z_t^1 = v_t,\quad z_t^{n+1} = x_t^n\,(h^n * z^n)_t,\ n=1,\dots,N,\quad y_t = z_t^{N+1} \]
— \(*\)는 (긴) convolution, \(x^n\)로 곱하는 것이 data-controlled gating.
🧮 복잡도 (Remark 3.1) ⭐Hyena recurrence는 \(O(N L \log_2 L)\). 입출력 맵 \(y = x^N\cdot(h^N * (x^{N-1}\cdot(h^{N-1} * (\cdots))))\), 각 convolution은 Fourier 도메인에서 \(O(L\log_2 L)\)(FFT). → attention의 \(O(L^2)\)보다 저렴.
🌍 실세계 시나리오
🌍 유전체는 수십만~수백만 bp(C16)라 self-attention의 \(O(L^2)\)면 메모리가 폭발한다. Hyena는 attention을 "긴 convolution+gating"으로 대체해 같은 전역 문맥을 \(O(L\log L)\)에 얻는다 → HyenaDNA가 1×A100·80분으로 학습 가능한 비결.
🔍 STEP-BY-STEP: Hyena 복잡도 \(O(L\log L)\) 유도 (난개념 · 9단계)
1. 직관
attention은 모든 위치쌍(L×L) 점수를 계산해 \(O(L^2)\). 긴 유전체에선 감당 불가 → 더 싼 전역 연산이 필요.
2. 셋업
핵심 연산을 "길이 L 필터 h와 길이 L 신호 z의 convolution \(h*z\)"로 둔다. 여기에 \(x\)로 elementwise 곱(gating)을 끼운다.
3. 전개 1
직접(시간 도메인) convolution은 각 출력마다 L곱 → 전체 \(O(L^2)\). 이러면 attention과 다를 바 없음 → 다른 방법 필요.
4. 전개 2
Convolution Theorem: 시간 도메인 convolution = Fourier 도메인 elementwise 곱. 즉 \(h*z = \text{IFFT}(\text{FFT}(h)\cdot\text{FFT}(z))\).
5. 전개 3
FFT/IFFT 각각 \(O(L\log_2 L)\), 중간 elementwise 곱은 \(O(L)\) → 한 convolution이 \(O(L\log_2 L)\).
6. 전개 4
Order-N operator는 이 convolution을 N번 인터리빙(gating과 함께) → \(O(N\,L\log_2 L)\). N은 상수 → 사실상 \(O(L\log_2 L)\).
7. 결론
attention \(O(L^2)\) → Hyena \(O(L\log_2 L)\) = subquadratic. L이 클수록 격차가 폭발적으로 벌어짐.
8. 검산/예
L=64K: \(L^2\approx 4\times10^9\) vs \(L\log_2 L\approx 64\text{K}\times16\approx10^6\) → 약 수천 배 차이(논문 "64K에서 100배 빠름"과 정합).
9. 시험 변형
"왜 FFT가 핵심인가"·"gating(\(x^n\) 곱)의 역할은"·"attention이 quadratic인 근본 이유는"을 단계별로 묻는다.
📊 성능 인사이트
long convolution+gating 인터리빙. 2K에서 학습 compute 20% 절감하며 Transformer 품질 도달, 8K에서 attention 대비 2배, 64K에서 100배 빠름.
⚠️ 함정
\(O(L\log L)\)의 log는 FFT 때문이지 트리 구조가 아님. gating(\(x\) 곱)은 데이터 의존 비선형성을 주는 부분(단순 convolution과의 차이).
📝 시험 포인트 ⭐⭐"왜 attention이 quadratic이고 Hyena가 subquadratic인가" = modeling 핵심. 서술형 대비: \(O(L\log L)\) 근거(Convolution Theorem + FFT)를 단계별로 유도. Notion이 Hyena를 특히 자세히 받아 적음.출처: 2.gLM.pdf p.21, ICML 2023

C19 아키텍처 비교: CNN vs Transformer vs Hyena ⭐高관문·수렴선행: C8, C11, C15 ▶
🎯 직관"local(CNN) ↔ global이나 \(O(L^2)\)(Transformer) ↔ global이면서 \(O(L\log L)\)(Hyena)."
📊 비교 인사이트 ⭐⭐ (통합 — 시험 종합판)
아키텍처연결 방식강점약점복잡도

🌍 실세계 시나리오
🌍 모델 선택 의사결정. "변이 주변 짧은 motif 탐지"면 CNN(NeuSomatic), "수백 bp 전역 문맥 이해"면 Transformer(DNABERT), "수십만 bp 초장문"이면 Hyena(HyenaDNA). 서열 길이와 local/global 요구가 아키텍처를 정한다.
🔍 STEP-BY-STEP: "어떤 모델을 왜 선택" 사고 흐름 (논문 해석형 대비 · 7단계)
1. 직관
시험은 종종 "이 태스크엔 어떤 아키텍처가 적합한가, 왜?"를 묻는다 → 3축으로 판단.
2. 축1: 서열 길이
짧다(수백 bp)→Transformer 무난. 매우 길다(수만~수백만)→\(O(L^2)\)가 막힘→Hyena.
3. 축2: local vs global
변이 주변 좁은 motif면 CNN으로 충분/효율적. 멀리 떨어진 regulatory 상호작용이면 global(Transformer/Hyena).
4. 축3: 이해 vs 생성
분류·예측(이해)→encoder/MLM(DNABERT). 생성·likelihood→decoder/CLM(GPT, Evo).
5. 결론
세 축을 조합해 모델 매핑: (긴+global+생성)→HyenaDNA/Evo, (짧은+global+이해)→DNABERT, (local+탐지)→CNN/NeuSomatic.
6. 검산/예
"1Mbp 유전체에서 enhancer-promoter 장거리 상호작용 예측" → 길이 큼+global+이해 → Hyena 계열 + (필요시 attention 하이브리드 Evo).
7. 시험 변형
비교표를 주고 "이 모델이 X를 택한 이유"·"\(O(L^2)\)의 의미"·"CNN을 안 쓴 이유"를 서술하게 한다.
⚠️ 함정
"Hyena가 무조건 최고"가 아님 — 짧은 서열·local 태스크엔 Transformer/CNN이 더 단순·충분. 복잡도는 trade-off지 우열이 아님.
📝 시험 포인트 ⭐⭐이 표가 "modeling 방법론" 출제 종합판. 서술형 대비: 각 아키텍처를 왜 선택하는지(서열 길이·local vs global·이해 vs 생성 3축)로 설명. \(O(L^2)\)의 출처(self-attention)도 함께.출처: 2.gLM.pdf p.11·20·21

라벨 없이 표현을 배우고(MLM/CLM), 대표 모델로 계보화. C11→C13→C20 · C12→C14 · C15/C12→C16→C17 · C10/C11→C18.

C10 사전학습 모델 (Pretraining) 개념 ⭐高관문선행: C4 ▶
🎯 직관"라벨 없는 방대한 서열에서 스스로 의미 있는 표현을 뽑는다 — 데이터가 많으면 라벨 없이도 패턴을 배운다."
📖 정의unlabeled genomic sequence에서 meaningful representation 추출, DNA의 구조·기능·진화적 특징을 포착.
🌍 실세계 시나리오
🌍 enhancer/promoter 라벨을 사람이 다 달려면 천문학적 비용. 대신 라벨 없는 유전체로 "가린 글자 맞히기"(MLM)만 시켜도 모델이 생물학적 문법을 배운다 → 이후 적은 라벨로 promoter 분류 같은 태스크를 빠르게 학습. Notion: "label 없어도 데이터가 많으면 패턴 인식 가능."
🔍 작동 원리 (2단계 도식)
  • (A) Self-supervised Pretraining: DNA 서열 토큰화→[CLS]…[SEP], 일부 [MASK] → gLM → Modeling Head → token probabilities (MLM/CLM).
  • (B) Supervised Fine-tuning: labeled DNA → gLM(pretrained weights, tunable 🔥 또는 frozen ❄️) → Task Head → 다운스트림(DNA: enhancer/promoter/insulator/splice site; RNA: mRNA expression/degradation/half-life).
✏️ 단계별 풀이 예제 — tunable vs frozen 선택 추론
1단계
라벨 데이터가 매우 적다(수백 개) → backbone를 다 풀면(tunable) 과적합 위험.
2단계
이때 backbone를 frozen ❄️ 하고 Task Head만 학습 → 적은 데이터로 안전.
3단계
라벨이 충분하고 도메인이 사전학습과 다르면 → tunable 🔥 로 전체 미세조정 → 성능 ↑.
답
데이터량·도메인 거리로 frozen/tunable을 고른다. (C21 실습은 인코더 tunable+분류기 학습.)
📊 표 인사이트
(B)의 다운스트림 예시들이 전부 "분류/회귀 head만 갈아끼우는" 구조 → 사전학습 표현의 재사용성.
⚠️ 함정
사전학습=비지도가 아니라 자기지도(self-supervised) — 라벨을 데이터 자체(가린 토큰)에서 만든다.
📝 시험 포인트 ⭐⭐자기지도학습이 "왜 라벨 없이 가능한가"(Notion 강조) + tunable vs frozen 차이. 서술형 대비: (A) 자기지도 사전학습과 (B) 지도 파인튜닝 2단계를 도식으로 그리고, frozen/tunable 선택 기준을 데이터량·도메인 거리로 설명. ProtBERT 실습이 바로 (B) 단계(C21).출처: Notion Day 2

C11 사전학습 ① Masked Language Modeling (MLM) ⭐高선행: C10 ▶
🎯 직관"문장 중간 글자를 가리고 좌우 문맥으로 맞히게 학습 — 양방향 이해에 강함."
📖 정의입력 토큰 일부를 무작위 [MASK] → 주변 문맥으로 마스크 토큰 예측. Encoder-based (Bidirectional Self-Attention).
🌍 실세계 시나리오
🌍 "ATG[MASK]CGT"에서 모델은 왼쪽(ATG)과 오른쪽(CGT)을 동시에 보고 빈칸을 추론한다. 양방향이라 promoter/enhancer 분류·splice site 예측처럼 서열을 이해하는 태스크에 강하다(DNABERT, C13).
🔍 작동 원리/효과upstream·downstream 양쪽 문맥에 의존하는 motif·signal 식별 → 임베딩 공간에서 생물학적으로 유사한 서열이 함께 클러스터링.
✏️ 단계별 풀이 예제 — MLM 마스킹 한 스텝 (seq = "A C T A C G")
1단계
토큰화 → [CLS] A C T A C G [SEP].
2단계
무작위로 ~15% 선택 → 예: 위치3 'T'를 고름.
3단계
선택 토큰을 [MASK]로 치환 → [CLS] A C [MASK] A C G [SEP].
4단계
인코더가 양방향으로 보고 [MASK] 자리 확률분포 출력 → 예 P(T)=0.7, P(A)=0.1….
5단계
정답 'T'와의 cross-entropy로 손실 → 역전파. (정답은 원본에서 가져옴 = 자기지도)
답
"가리고→양방향 문맥으로→맞히고→손실"의 한 사이클.
📊 표 인사이트
슬라이드 요약 "See the entire sequence (left+right) → Strong at understanding."
⚠️ 함정
MLM의 [MASK]는 사전학습 때만 존재. 파인튜닝·추론 시엔 마스크 없음(train-test 불일치 이슈는 BERT 계열 공통).
📝 시험 포인트 ⭐MLM=양방향=이해 ↔ CLM=단방향=생성 대비(C12). 서술형 대비: 인코더·양방향 self-attention이 왜 "이해" 태스크에 유리한지 설명. DNABERT가 MLM 기반(C13).출처: Notion Day 2
🔗 연결선행 ← C10 / 대조 → C12 / 응용 → C13, C18, C20
C12 사전학습 ② Causal Language Modeling (CLM) ⭐高선행: C10, C11 ▶
🎯 직관"이전 토큰들로 다음 토큰을 맞히게 학습 — 서열 생성과 likelihood 계산에 강함."
📖 정의모든 이전 토큰이 주어졌을 때 다음 토큰 예측(autoregressive). Decoder-based (Causal Self-Attention), 단방향.
🌍 실세계 시나리오
🌍 Evo(C17)는 CLM으로 새로운 CRISPR-Cas 단백질·유전체 서열을 생성한다. 또 변이 서열의 likelihood를 계산해 "이 변이가 자연스러운가(=병원성 낮음)"를 점수화(variant scoring) — 생성·확률평가가 CLM의 두 강점.
🔍 작동 원리/효과토큰 단위 서열 생성, 서열에 대한 확률분포 학습.
✏️ 단계별 풀이 예제 — CLM 다음토큰 예측 + likelihood (seq = "A C T")
1단계
위치1 입력 [CLS] → 다음 토큰 예측, 정답 A. P(A)=0.5.
2단계
위치1~2 입력 [CLS]A → 다음 예측, 정답 C. P(C|A)=0.4. 오른쪽은 절대 못 봄(causal mask).
3단계
위치1~3 입력 [CLS]A C → 다음 예측, 정답 T. P(T|AC)=0.6.
4단계 · likelihood
서열 likelihood = P(A)·P(C|A)·P(T|AC) = 0.5·0.4·0.6 = 0.12 (또는 log합).
답
autoregressive 곱으로 서열 확률을 얻음 → variant scoring의 원리.
📊 표 인사이트
슬라이드 요약 "See only the past (left context) → Strong at generation & likelihood."
⚠️ 함정
CLM은 미래 토큰을 causal mask로 가린다 — MLM처럼 양방향이 아님. "이해" 태스크엔 MLM이 보통 유리.
📝 시험 포인트 ⭐MLM vs CLM "양방향/단방향 → 이해/생성" 차이 핵심 비교. 서술형 대비: 디코더·causal mask가 왜 생성·likelihood에 강한지, autoregressive 확률 곱으로 variant scoring 원리 설명. Nucleotide GPT·Evo가 CLM 계열(C14, C17).출처: Notion Day 2
🔗 연결선행 ← C10, C11 / 대조 → C11 / 응용 → C14, C16, C17
C13 DNABERT ⭐高선행: C6(k-mer), C11(MLM) ▶
🎯 직관"BERT를 DNA에 — k-mer 토큰 + MLM 사전학습 + 태스크별 파인튜닝."
📖 정의/구조Encoder-based transformer (12× 블록: Multi-Head Self-Attention → Add&LayerNorm → Feed Forward → Add&LayerNorm). 입력 = Token Embedding + Positional Embedding. 원서열을 k-mer로 tokenize([CLS] AGC GCA CAC … [SEP]), pretrain 시에만 [MASK].
📖 출력 헤드Sentence-level classifier(원 서열 분류) + Token-level classifier(각 마스크 토큰 분류).
🌍 실세계 시나리오
🌍 스플라이스 부위(intron 제거 경계, 5' donor GU/GC·3' acceptor AG)를 정확히 찾으면 alternative splicing으로 한 유전자가 만드는 여러 단백질을 예측할 수 있다. DNABERT는 양방향(MLM) 문맥으로 canonical·non-canonical splice 신호를 잡아 기존 도구를 능가.
📊 표·그래프 인사이트 (Splice site, p.17) ⭐
DNABERT-Splice가 Accuracy 0.92 / F1 0.91로 1위 (SpliceFinder 0.833/0.826 > GBM > LR > LSTM > GRU > DBN > RF > tree > SVM_RBF 순 하락). → 양방향 문맥으로 splice 신호를 더 잘 포착.
✏️ 단계별 풀이 예제 — 막대그래프 해석(논문 해석 연습)
1단계
x축=모델, y축=Accuracy/F1. DNABERT-Splice가 0.92로 최상단.
2단계
딥러닝 baseline(LSTM/GRU 0.8대) vs DNABERT 격차 → "양방향 인코더 + 사전학습" 효과로 해석.
3단계
전통 ML(RF, SVM)은 더 낮음 → feature engineering의 한계(C3와 연결).
답
"DNABERT 우위 = MLM 양방향 문맥이 splice donor/acceptor 신호를 포착했기 때문."
⚠️ 함정
DNABERT의 [MASK]는 사전학습 전용. 12× 블록은 BERT-base 구조와 동일.

C14 Nucleotide GPT ⭐高선행: C5, C12(CLM) ▶
🎯 직관"GPT를 DNA에 — single-nucleotide 토큰 + 디코더(CLM)로 사전학습, 사전학습이 표현에 기여하는지 probing."

🌍 실세계 시나리오
🌍 "사전학습이 정말 도움이 되나?"를 증명하려는 연구. 같은 구조를 (Linear/one-hot) vs (Random init) vs (Pretrained)로 비교해 → Pretrained가 압도적임을 Table로 보임 → C10의 "데이터 많으면 패턴 학습" 주장을 정량 입증.
🧮 수식 (RE loss weighting)
\[ L = -\sum_{i=1}^{n} w_i \cdot y_i \log(\hat{y}_i) \]
\(w_i\)=클래스/태스크별 가중, \(y_i\)=정답(one-hot), \(\hat{y}_i\)=예측확률. 왜 weighting? → 태스크 불균형·보조신호 반영.
📊 표 인사이트 (시험 빈출 ⭐⭐, 논문 해석)

  • Table 3 (6-mer overlap / 6-mer non-overlap / single-nucleotide): single-nucleotide 대체로 최고(Coding vs intergenic 92.4). 일부(Ensembl regulatory)는 6-mer overlap 우세 → 태스크 의존.

1단계

3단계

답
격차를 "구조 기여(Random−Linear)"와 "사전학습 기여(Pretrained−Random)"로 분해해 설명.

C17 Evo (StripedHyena, Hybrid) ⭐中선행: C15, C16 ▶
🎯 직관"Hyena 블록 사이에 Rotary Attention을 섞은 하이브리드(StripedHyena)로 분자~유전체 스케일을 통합 모델링·생성."
📖 정의/구조 (p.27 B)StripedHyena = 다수 Hyena 블록 + 간헐적 Rotary Attention 블록 인터리빙. Hyena 블록 내부 = Dense → Convolution → Gate → Long Convolution → Gate → Convolution → GLU (Sequence Mix=파랑, Channel Mix=빨강). 출력 = next-base likelihood \(p(x_N)\).
🌍 실세계 시나리오
🌍 Evo는 수백만 prokaryotic·phage genome을 학습해 CRISPR-Cas·transposon을 새로 생성(protein-RNA, protein-DNA codesign 첫 사례)하고, small mutation→fitness를 zero-shot 예측한다. Evo 2는 "all domains of life". 순수 Hyena의 효율 + 간헐적 attention의 정밀 문맥을 둘 다 취하려는 설계.
🧮 수식 (LoRA 삽입)Attention·FFN·Hyena long-convolution filter layer에 low-rank adapter 삽입.
\[ h = W_0 x + \Delta W x = W_0 x + BA x \]
\(W_0\in\mathbb{R}^{d\times d}\)(frozen pretrained), \(B=0\) 초기화, \(A\sim\mathcal{N}(0,\sigma^2)\), rank \(r\ll d\).
✏️ 단계별 풀이 예제 — LoRA 식 해석 "왜 B=0 초기화?"
1단계
파인튜닝 시작 시 \(\Delta W = BA\). \(B=0\)이면 \(\Delta W=0\).
2단계
따라서 시작 시 \(h = W_0 x\) → 사전학습 모델과 완전히 동일(망가뜨리지 않음).
3단계
학습이 진행되며 \(B\)가 0에서 멀어져 점진적으로 적응.
4단계
\(r\ll d\)라 추가 파라미터 \(2dr\)만 학습 → 적은 비용 파인튜닝.
답
"B=0은 안전한 출발점(사전학습 보존), low-rank는 효율"이 LoRA 핵심.
⚠️ 함정
LoRA는 \(W_0\)를 freeze하고 \(BA\)만 학습 — 원 가중치를 직접 바꾸지 않는다.
📝 시험 포인트 ⭐하이브리드(Hyena+attention) 동기 + LoRA 식 \(h=W_0x+BAx\) 해석(B=0 이유). 서술형 대비: 왜 순수 Hyena가 아닌 하이브리드인지, B=0 초기화·\(r\ll d\)가 각각 무엇을 보장하는지.출처: 2.gLM.pdf p.26–28, Science 2024 / Evo2 Nature 2026
🔗 연결선행 ← C15, C16 / 비교 → C19
C18 ESM (Evolutionary Scale Modeling, 단백질 LM) ⭐中선행: C10, C11 ▶
🎯 직관"단백질 1차서열만으로 LM이 원자 수준 3D 구조를 직접 예측 — 모델을 키우면 구조가 표현에서 '창발'."
📖 정의/결과단백질 서열 LM을 15B 파라미터까지 키우면 표현에서 atomic-resolution 구조가 emerge. >617M metagenomic 단백질 구조 예측(ESM Metagenomic Atlas), MSA 없이 1차서열만으로 직접 추론 → 고해상도 구조예측을 order-of-magnitude 가속.
🌍 실세계 시나리오
🌍 AlphaFold는 MSA(다중서열정렬)가 필요해 느리다. ESM은 서열 하나만 넣어도 구조를 추론 → 신약 타깃의 막관통 영역·결합 포켓을 빠르게 예측. 본 과목 실습(C21, 단백질 막관통 분류)이 바로 이 "단백질 LM" 계열.
🧮 수식 (RMSD)
\[ RMSD = \sqrt{\dfrac{1}{N}\sum_{i=1}^{N}\lVert x_i^{pred} - x_i^{true}\rVert^2} \]
예측·실제 원자좌표 차의 RMS, 작을수록 정확(주석 "2–5: 상당히 유사함").
📊 그래프 인사이트
ESM-2를 8M→15B로 키울수록 RMSD↓(7.7→2.6), pLDDT↑(신뢰도), Perplexity↓ → 스케일↑ = 구조예측 품질↑.
✏️ 단계별 풀이 예제 — RMSD 계산 (3개 원자)
1단계
예측-실제 좌표 차 벡터 노름² 구하기 → 원자1 차²=4, 원자2=1, 원자3=9 (예시값).
2단계
평균 → (4+1+9)/3 = 4.67.
3단계
제곱근 → √4.67 ≈ 2.16 Å.
4단계
주석 기준 "2–5 상당히 유사" → 꽤 정확한 예측으로 해석.
답
RMSD≈2.16 Å. 스케일 큰 ESM일수록 이 값이 작아짐.
⚠️ 함정
"창발(emerge)"은 명시 학습 목표가 아니라 스케일의 부산물 — 구조 라벨로 학습한 게 아니라 서열 LM만 했는데 구조가 표현에 잡힘.
📝 시험 포인트 ⭐"왜 LM 스케일↑이면 구조가 나오나"(표현학습), RMSD·pLDDT·perplexity 지표 해석. 서술형 대비: 세 지표가 각각 무엇을 측정하는지, 스케일과의 단조 관계를 그래프로 설명. 실습 ProtBERT와 같은 단백질 LM 계열(C21).출처: 2.gLM.pdf p.29
🔗 연결선행 ← C10, C11 / 응용 → C21(실습)
C20 VIBE & 바이러스 분류 (응용) ⭐低선행: C13(BERT 인코더) ▶
🎯 직관"메타게놈 샷건 reads를 Transformer로 바이러스 분류 — 미지 바이러스에도 높은 recall."
📖 메커니즘marker 기반·homology 검색은 known virus엔 좋지만 unknown엔 recall 낮음 → 일반화 모델로 recall↑. VIBE = forward/backward read를 [CLS]…[SEP](max 512)로, MLM 사전학습, 12× Transformer encoder + classification(softmax, confidence). 계층적: Domain(BPDR) → Bacteria/Phage/DNA·RNA virus → Order → lower-level.
🌍 실세계 시나리오
🌍 임상 메타게놈 샘플에서 정체불명 바이러스 reads를 분류해야 하는데, homology 검색은 DB에 없는 신종을 놓친다. VIBE는 학습 기반 일반화로 미지 종도 잡아 → HVP 환자에서 co-infection(HPV=Alphapapillomavirus, Herpesvirales) 탐지.
📊 그래프 인사이트
genome-level validation(테스트 종 전부 학습 제외)에서 VIBE가 CHEER/Kraken2 대비 recall 우위. fine-tuned 임베딩(C)이 4-mer frequency(A)보다 코로나바이러스 속을 명확히 분리.
✏️ 단계별 풀이 예제 — "왜 homology보다 recall이 높나" 추론
1단계
homology 검색 = 쿼리를 DB 서열과 정렬해 유사도로 판정.
2단계
DB에 없는 신종은 정렬 실패 → 놓침(낮은 recall).
3단계
VIBE는 사전학습으로 "바이러스다움"의 일반 패턴을 학습 → 정확한 종 매칭 없이도 분류.
4단계
test 종을 학습에서 전부 제외해도 recall 유지 → 일반화 입증.
답
"정렬 매칭(DB 의존) vs 학습 일반화"의 차이가 recall 격차를 만든다.
⚠️ 함정
VIBE의 강점은 정확도 전반이 아니라 특히 unknown에 대한 recall.
📝 시험 포인트 ⭐"homology 검색 한계 vs 학습 기반 일반화"의 recall 개선 논리. 서술형 대비: 정렬 기반 방법이 신종에서 왜 recall이 낮은지, 학습 기반이 어떻게 이를 극복하는지 서술. 응용 예시 출제 가능.출처: 2.gLM.pdf p.33–35
🔗 연결선행 ← C13 / 동기 → C2(변이/서열 다양성)

B5. 전이학습 실습 · 평가

사전학습 모델을 가져와 분류기만 붙여 파인튜닝·평가. C4 + C10 + C18의 실체화.

C21 【실습】 단백질 시퀀스 이진분류 (ProtBERT 전이학습) ⭐高종착선행: C4, C10, C18 ▶
🎯 직관"사전학습된 단백질 LM(ProtBERT)을 가져와, 마지막에 분류기만 붙여 '막관통 단백질인지'를 맞히는 파인튜닝."
📖 태스크 정의입력=아미노산 서열, 출력=이진분류 0(non-membrane)/1(membrane). 데이터=DeepLoc 2.0의 membrane 라벨(10개 위치 다중분류 중 membrane 여부만 이진화).
🌍 실세계 시나리오
🌍 막관통 단백질은 신약 타깃의 약 절반(GPCR 등). 어떤 단백질이 세포막을 관통하는지 서열만으로 예측하면 신약 타깃 발굴이 빨라진다. 라벨 데이터(2,700개)는 적지만, 거대 사전학습 ProtBERT를 재사용해 적은 데이터로 학습 — C10(B) 파인튜닝의 실체.
📖 데이터 스키마 (CSV, 직접 확인) ⭐
  • 컬럼: index, acc, sequence, labels. acc=UniProt accession(예 P63086), sequence=20-letter 아미노산(M 시작 다수), labels=0/1.
  • train=2,700행(0:1350/1:1350, 완전 균형 50:50), valid=300행(0:150/1:150, 균형).
  • 길이: train min 40/max 8,886/중앙값 440/평균≈562 aa → 가변·일부 매우 긺 → max_length=512 truncation.
🔍 작동 원리 (딥러닝 4단계)Step1 데이터 → Step2 모델 → Step3 학습 → Step4 평가(AUROC).
🔍 STEP-BY-STEP: 서열 → 토크나이징 → 임베딩 → pooling → 분류 파이프라인 (실습 핵심 · 10단계)
1. 직관
"가변 길이 단백질 서열 1개"를 "고정 길이 벡터 1개 → 0/1 라벨"로 바꾸는 전 과정.
2. Step1 전처리
re.sub(r"[UZOB]","X",seq)로 희귀/모호 아미노산(U,Z,O,B)→X 치환 후 글자 사이 공백 join(ProtBERT 입력 형식). Dataset/DataLoader(shuffle=True for train).
3. Step2 토크나이즈
BertTokenizer.batch_encode_plus(add_special_tokens=False, max_length=512, padding=True, truncation=True) → 길이 512로 통일(긴 건 자르고 짧은 건 패딩).
4. Step2 임베딩
BertModel.from_pretrained("Rostlab/prot_bert")(인코더)에 통과 → 각 아미노산 위치마다 임베딩 벡터 → shape (batch, 512, hidden).
5. Step2 pooling
pooled = torch.mean(embeddings[0], dim=1) — 512개 위치 임베딩을 평균(mean pooling) → protein-level 단일 벡터 (batch, hidden).
6. Step2 분류기
Dense classifier(Linear→ReLU→Dropout×num_layers, 마지막 Linear(hidden, 2)) → logits (batch, 2).
7. Step3 학습
CrossEntropyLoss + Adam(lr=0.001). Loop: forward→loss.backward()→optimizer.step()→zero_grad(). (하이퍼: hidden 64, layers 3, batch 64, epochs 1, dropout 0.0)
8. Step4 평가
model.eval()+torch.no_grad()(gradient 비활성→메모리·속도)로 valid forward, loss·accuracy(가중치 업데이트 없음), AUROC.
9. 검산/예
입력 서열 "M A K … L"(길이 600) → X치환·공백join → encode(512로 truncation) → 임베딩(512×hidden) → mean → (hidden,) → Linear → logits [−0.3, 1.2] → softmax → P(membrane)=0.82 → 라벨 1.
10. 시험 변형
"mean을 max/attention으로 바꾸면?"(아래 Pooling), "CrossEntropy를 BCE로?"(아래 Loss), "왜 max_length=512인가"(길이 분포·메모리).
📊 표·그래프 인사이트 ⭐
학습곡선 — Train loss 계속 하강·Train acc↑이지만 Validation loss ~10epoch 이후 다시 상승(U자) → 과적합(overfitting) 신호. Valid acc는 0.78~0.82 정체. (dropout=0.0, lr=0.001로 정규화 약함이 원인 후보.)
✏️ 단계별 풀이 예제 — Pooling 전략 비교 (Notion 질문, 시험 직결)
1단계 · mean pooling (현재)
512위치 임베딩 평균. 전체를 고루 반영하나 신호가 희석될 수 있음.
2단계 · max pooling
위치별 최댓값만. 가장 강한 motif 강조 ↔ 노이즈에 취약.
3단계 · attention pooling
위치별 학습 가중합. 막관통 신호가 있는 잔기를 자동 강조.
4단계 · 판단
membrane 신호가 특정 막관통 helix 잔기에 국소화되면 attention/max가 mean보다 유리할 수 있음.
답
"어떤 잔기가 membrane 신호인지"에 따라 pooling 선택이 성능을 가른다.
⚠️ 함정
(1) 데이터는 균형(50:50)이라 accuracy가 유효 지표지만, 실무 불균형이면 AUROC/F1 필요. (2) epochs=1·dropout=0 설정은 과적합 관찰용 — 정규화(dropout↑, lr↓)로 완화 가능.
📝 시험·실습 포인트 ⭐⭐
  • Pooling(Notion 질문): mean vs max vs attention 차이 → 위 예제. 서술형 대비: 각 pooling이 어떤 신호를 강조/희석하는지.
  • Loss 변경(Notion Tip): 2-class softmax+CrossEntropy ↔ 단일 logit+sigmoid+BCE. 이진분류에서 거의 등가지만 출력 헤드·임계값 처리가 다름.
  • 전이학습 핵심: pretrained 인코더(frozen/tunable)+가벼운 분류기 → 적은 라벨로 학습 = C10(B)의 실체.
