B1. 탐색 — Uninformed (정보 없는 탐색)

C1 · 탐색 문제 정식화 & 평가 4지표 ⭐高선행: 없음(시작점)관문▶
🎯 한 줄 직관상태=노드, 연산자=엣지인 그래프를 탐색트리로 펼쳐 목표를 찾는다.
📖 정의search space = states + operators(그래프). search tree = 그 그래프의 한 탐색 전개. search strategy = 노드 확장 순서의 선택.
🔍 평가 4지표 + 3대 파라미터
  • Completeness: 해가 존재하면 반드시 찾는가?
  • Optimality: 항상 최소비용(또는 최소깊이) 해를 찾는가?
  • Time complexity: 생성된 노드 수(worst case)
  • Space complexity: 메모리에 동시에 올라가는 최대 노드 수
b=최대 분기수(branching factor), d=최소비용해의 깊이, m=상태공간 최대 깊이(∞ 가능).
📊 인사이트복잡도는 "생성된 노드 수" 기준 → Min-Heap 정렬 같은 부가비용은 평가에서 무시(Day1 교수 강조 질문). b가 커지면 지수 폭발이 본질.
⚠️ 함정
time과 space를 혼동. space는 "동시에 메모리에 올라가는 최대" — 뒤의 BFS 지수공간/DFS 선형공간 논쟁의 기준.
📝 시험 포인트 (서술형)"4지표(완전·최적·시간·공간)를 각각 정의하고, b/d/m이 무엇을 뜻하는지 설명하라." → 네 단어 정의 + 세 파라미터 의미를 자기 말로. 서술형 단골.
🔗 연결응용 → C2·C3·C6·C11·C14 (모든 탐색 줄기의 뿌리)
C2 · BFS (Breadth-First Search) ⭐高선행: C1▶
🎯 한 줄 직관가장 얕은 미확장 노드 먼저 = 레벨 순서로 펼친다.
📖 정의 / 자료구조FIFO Queue(새 successor를 큐 끝에 추가). ⚠️ 정정 학생 노트(Notion Day1)는 이를 Stack으로 잘못 표기 — 정답은 FIFO Queue.
📊 표·그래프 인사이트
지표값왜
CompleteYes유한 b면 결국 모든 레벨 도달
OptimalYes (단계비용 동일 가정)얕은 해 먼저
TimeO(b^d) (정밀히 O(b^{d+1}))깊이 d까지 노드 누적
SpaceO(b^d)지수 메모리 = 치명적 약점
정량 예(b=10, 10⁴ node/s, 1KB/node): d=8 → ≈10⁹노드/≈31시간/1TB; d=12 → ≈10¹³노드/≈35년/10PB. → BFS의 실질 한계는 시간보다 메모리(지수 공간).
⚠️ 함정
"Time이 약점"이라 답하기 쉬우나 슬라이드 강조는 exponential SPACE(메모리).
📝 시험 포인트 (서술형)"BFS의 자료구조와 4지표를 말하고, 왜 실무에서 못 쓰는지 설명하라." → FIFO Queue · 완전·최적 · 지수 공간(메모리) 때문. 정량 예(10PB)를 근거로 들면 강함.
🔗 연결선행 C1 / 대조 C3(DFS, 메모리 정반대) / 일반화 C5(UCS)
C3 · DFS (Depth-First Search) ⭐高선행: C1▶
🎯 한 줄 직관가장 깊은 미확장 노드 먼저, 막히면 backtrack.
📖 정의 / 자료구조LIFO Stack(새 successor를 스택 앞에 추가). ⚠️ 정정 학생 노트 뒤바뀜 — 정답은 LIFO Stack.
📊 표·그래프 인사이트
지표값왜
CompleteNo (트리 깊이 무한이면)무한 경로에 빠질 수 있음
OptimalNo먼저 찾은 해가 최소가 아닐 수 있음
TimeO(b^m)최악 전체 깊이 탐색
SpaceO(bm) = Linear깊이 깊어져도 메모리 일정 = 최대 장점
BFS vs DFS: Time은 같은 꼴이나 — BFS는 무한경로/많은 loop/얕은 해에 유리; DFS는 메모리 선형(압도적 우위), 해가 많고 loop 적고 무한경로 없을 때 유리.
⚠️ 함정
DFS의 핵심 가치는 "빠름"이 아니라 선형 공간. 비완전성은 "무한 깊이"라는 조건에서만.
📝 시험 포인트 (서술형)"DFS의 자료구조와 가장 큰 장점, 그리고 언제 불완전한지를 설명하라." → LIFO Stack · 장점=선형 공간 O(bm) · 무한 깊이 트리에서 불완전.
🔗 연결선행 C1 / 대조 C2(BFS) / 응용 C4(IDS), C14·C15(backtracking=DFS)
C4 · Depth-Limited & IDS ⭐高선행: C2, C3▶
🎯 한 줄 직관DFS를 depth-limit L을 1씩 키우며 반복 → DFS의 선형 메모리 + BFS의 완전성·최적성.
📖 정의Depth-Limited(L): 무한경로 문제는 해결하나, 해가 L보다 깊으면 불완전. L은 문제 지식(예: state-space 지름)으로 선택.
IDS 의사코드: L=1; while no solution: DFS(initial, cutoff=L); if found return; else L++
📊 인사이트 ("낭비 아닌가?")
반복 탐색이지만 대부분 시간은 가장 깊은 레벨 d에서 소모 → BFS 대비 약 11%만 더 듦 (b=10,d=5: N(IDS)≈123,450 vs N(BFS)≈111,110, 비율 ≈ b/(b−1)). 성질: Space O(bd), Time O(b^d), Complete Yes, Optimal Yes(경로비용이 깊이의 비감소 함수일 때).
✏️ 단계별 풀이 예제
IDS가 "낭비 아님"을 수로 보이기
1. 직관
IDS는 L=1,2,3…로 DFS를 반복하니 윗 레벨을 여러 번 본다 — 손해 같다.
2. 셋업
b=10(분기), 해 깊이 d=5. 노드 수는 레벨이 깊을수록 b배씩 폭증.
3. 전개1·왜
레벨 0 노드는 d+1번, 레벨1은 d번… 가장 깊은 레벨 d는 1번만 생성. 즉 많이 반복되는 건 노드가 적은 윗레벨.
4. 전개2
계산: N(IDS)≈123,450, N(BFS)≈111,110.
5. 결론
비율 ≈ b/(b−1) = 10/9 ≈ 1.11 → 11%만 더 듦. 메모리는 O(bd)로 선형.
6. 검산
b가 클수록 b/(b−1)→1 → 분기 클수록 낭비 더 적음.
7. 시험 변형
"IDS의 시간이 BFS보다 약간 더 드는데도 표준으로 쓰는 이유?" → 메모리 선형 + 완전·최적, 추가 시간은 상수배(약 11%)에 불과.
⚠️ 함정
"재탐색하니 훨씬 느리다"는 오해 — 지수 트리에서 마지막 레벨이 노드 수를 지배하므로 재방문 비용은 작다.
📝 시험 포인트 (서술형)"IDS는 왜 좋은가? 반복하는데 왜 낭비가 아닌가?" → DFS 메모리(선형) + BFS 완전·최적 결합, 상위 레벨 재탐색 비용은 마지막 레벨에 비해 미미(약 11%). 큰 공간 + 미지의 해 깊이일 때 uninformed 표준.
🔗 연결선행 C2·C3 / 종합 C5b
C5 · UCS (Uniform Cost Search) ⭐高선행: C2▶
🎯 한 줄 직관g(n)=시작~n 경로비용이 최소인 fringe 노드를 항상 확장 (BFS의 비용 일반화).
📖 정의 / 자료구조Min-Heap (priority queue). (학생 노트도 여기는 Min-Heap으로 맞게 표기.)
🧮 수식\( g(n) \) = 시작 상태 S에서 노드 n까지의 누적 경로비용. 항상 min g(n) 노드를 확장.
🔍 작동 원리 / 최적성·완전성 (PDF 명시)
  • 가정: 모든 step 비용 ≥ ε > 0.
  • Completeness: ε>0 + 유한 b ⇒ 목표 비용 도달까지 유한 확장 ⇒ 유한 단계 내 도달.
  • Optimality(모순법): UCS가 비최적이라 가정 → 더 작은 비용의 목표 존재 → 그러나 UCS는 정의상 그 노드를 먼저 확장했을 것 → 모순.
📊 인사이트
복잡도 시간·공간 \( O(b^{1+\lfloor C^*/\varepsilon \rfloor}) \) (C*=최적해 비용). 모든 비용이 같으면 BFS와 동일 동작.
⚠️ 함정
"UCS는 항상 BFS와 다르다"는 오해 — 비용이 균일하면 BFS와 같다.
📝 시험 포인트 (서술형)"UCS는 무엇을 우선 확장하며, 왜 최적인가?" → min g(n) 확장, 모순법(더 싼 목표가 있었다면 먼저 확장됐어야 함). 조건 = ε>0·비음수 비용. 증명 손계산은 안 나오나 모순법 골자를 말로 설명할 수 있어야.

B2. 탐색 — Informed & Local (정보·지역 탐색)

C6 · Informed Search & Heuristic Function ⭐高선행: C1관문▶
🎯 한 줄 직관문제별 휴리스틱 h(n)으로 "목표에 가까워 보이는" 노드를 우선 → 평균 속도 대폭 향상.
📖 정의Best-first search = 평가함수 f(n)("desirability")이 낮은 노드 우선 확장. h(n)=n→goal 추정비용("rule of thumb"), h(goal)=0. 예: 직선거리(SLD).
🧮 8-puzzle 두 휴리스틱h₁=잘못 놓인 타일 수(misplaced tiles), h₂=타일들의 맨해튼 거리 합. 예시 상태 h₁=8, h₂=18.
🌍 실세계 시나리오
내비게이션 — 목표 도시까지의 직선거리(SLD)를 h로 쓰면 실제 도로를 다 펼치지 않고 "목표 방향" 도로를 먼저 탐색 → 8-puzzle처럼 평균 탐색량이 수십~수백 배 준다.
⚠️ 함정
h는 "보이는" 추정일 뿐 실제비용이 아님 → 그래서 다음 노드(Greedy)가 함정에 빠질 수 있음.
📝 시험 포인트 (서술형)"휴리스틱 함수란? best-first의 일반형 f(n)은? 8-puzzle h₁·h₂는 어떻게 계산하나?" → 정의(목표까지 추정비용, h(goal)=0) + h₁=오배치 타일 수, h₂=맨해튼 합.
🔗 연결선행 C1 / 응용 C7·C8·C10
C7 · Greedy Best-First Search ⭐中선행: C6▶
🎯 한 줄 직관f(n)=h(n)만 사용 — "목표에 가장 가까워 보이는" 노드만 확장.
📖 성질Complete? No(상태 추적 안 하면 loop, DFS처럼 막힘) / Optimal? No / Time·Space O(b^m).
🌍 실세계 시나리오
루마니아 Arad→Bucharest. Greedy는 매번 직선거리가 짧아 보이는 도시로만 가다 Fagaras 경유의 비최적 경로를 택한다 — "가까워 보이는 것"이 "총 거리가 짧은 것"은 아니기 때문.
✏️ 단계별 풀이 예제 (Arad→Bucharest, h만 보기)
Greedy가 Fagaras 함정에 빠지는 과정
1. 직관
f(n)=h(n)만 사용 — 목표(Bucharest)까지 직선거리(SLD)가 가장 짧아 보이는 도시로만 전진. 지나온 비용 g는 안 본다.
2. 셋업
h=직선거리: Arad 366, Sibiu 253, Timisoara 329, Zerind 374, Fagaras 176, Rimnicu Vilcea(RV) 193, Pitesti 100, Bucharest 0. 실제 도로비용 g는 따로(예: Sibiu–Fagaras 99, Sibiu–RV 80…).
3. 전개1
Arad 확장 → 이웃 Sibiu(253)·Timisoara(329)·Zerind(374). h 최소 Sibiu(253) 선택.
4. 전개2 · 함정 발생
Sibiu 확장 → 이웃 Fagaras(176)·RV(193)·Arad(366)·Oradea(380). h 최소라 Fagaras(176) 선택 — RV(193)가 17 크다고 버린다(여기가 함정).
5. 전개3
Fagaras 확장 → Bucharest(0) 도달, 종료. 경로 = Arad→Sibiu→Fagaras→Bucharest.
6. 결과(실제비용)
g 합 = 140(A–S) + 99(S–F) + 211(F–B) = 450.
7. 검산 · 왜 비최적
최적해는 Arad→Sibiu→RV→Pitesti→Bucharest = 140+80+97+101 = 418. Greedy는 g를 무시하고 h만 봐 Fagaras(176<193)를 택했지만, 총거리는 RV 경로가 32 짧다.
8. 시험 변형
같은 그래프를 A*(f=g+h)로 풀면 RV–Pitesti 경로(418)를 찾는다 → C8과 대조해 "왜 h만으론 안 되나"를 설명.
⚠️ 함정
g(n)(지나온 비용)을 무시 → 가까워 보여도 실제 비용 큰 경로 선택.
📝 시험 포인트 (서술형)"Greedy가 비최적인 이유를 한 문장으로." → 누적비용 g(n)을 무시하고 h만 보기 때문. (A*와의 대조로 묻기 좋음.)
🔗 연결선행 C6 / 대조 C8(A*는 g+h)
C8 · A* Search ⭐高선행: C6, C7▶
🎯 한 줄 직관f(n)=g(n)+h(n) — 지나온 비용 + 남은 추정비용을 합쳐 균형.
🧮 수식\( f(n)=g(n)+h(n) \). g(n)=시작~n 실제비용, h(n)=n~goal 추정비용.
📖 성질Complete Yes / Optimal Yes (admissible h, TREE-SEARCH) / optimally efficient(같은 h에서 어떤 최적 알고리즘도 A*보다 적게 확장 못함) / 최악 지수.
A 알고리즘 vs A*: 차이 = admissible heuristic 사용 여부. 최적성 보장에 엣지 비용 ≥ 0(음수 불가) 필요.
🌍 실세계 시나리오
내비 경로탐색 — g=이미 달린 거리, h=목적지까지 직선거리. 둘을 더해 "지금까지+앞으로"가 최소인 도로를 확장 → Greedy의 Fagaras 함정을 피하고 실제 최단경로를 보장.
✏️ 단계별 풀이 예제
8-puzzle을 A*로 한 수 전개
1. 직관
빈 칸으로 인접 타일을 밀어 목표 배치를 만든다. 각 후보 수를 f=g+h로 점수 매겨 가장 낮은 것을 먼저.
2. 셋업
g=지금까지 민 횟수, h=h₂(각 타일의 맨해튼 거리 합). 시작상태 예: h₂=18.
3. 전개1
시작 노드 확장 → 빈 칸이 만드는 합법수(예 3개) 각각의 자식 생성. 각 자식 g=1.
4. 전개2·왜
자식마다 h₂ 재계산(타일 하나만 움직였으니 그 타일 거리만 ±1). f=1+h₂.
5. 전개3
세 자식 중 f가 가장 작은 노드를 fringe(min-heap)에서 꺼내 다음 확장. (h₂가 16이면 f=17이 최저.)
6. 결론
이 과정을 반복하면 A*는 목표까지 f 최소 경로를 따라가며, admissible h₂ 덕에 첫 목표 도달이 곧 최적해.
7. 검산
h₂는 절대 실제 이동수를 넘지 않음(타일당 최소 이동만 셈) → admissible → 최적 보장.
8. 시험 변형
"왜 h₂(맨해튼)가 h₁(오배치 수)보다 노드를 적게 확장하나?" → C9 dominance로 연결.
⚠️ 함정
h만(Greedy)도 g만(UCS)도 아닌 합이 핵심. h가 과대평가하면 최적성 깨짐(→C9).
📝 시험 포인트 (서술형)"f=g+h의 각 항 의미와, A*가 최적이 되는 조건은?" → g=실제 누적, h=추정 잔여, 조건=admissible. "A*가 optimally efficient라는 말의 뜻?"도 단골.
🔗 연결선행 C6·C7 / 심화 C9(admissible·dominance) / 대조 C5(UCS=h≡0인 A*)
C9 · Admissibility · Dominance · Relaxed Problems ⭐高선행: C8▶
🎯 한 줄 직관"좋은 휴리스틱"이란 절대 과대평가하지 않으면서(admissible) 실제값에 최대한 가까운(dominant) 것.
📖 정의
  • Admissible(허용성): 모든 n에서 \( h(n) \le h^*(n) \) (h*=실제 최적비용). = 과대평가 안 함 = optimistic. 예: SLD는 실제 도로거리를 절대 초과 안 함. 정리: h admissible ⇒ A*(TREE-SEARCH) optimal.
  • Dominance(우월): admissible h₁,h₂에서 모든 n에 \( h_2(n) \ge h_1(n) \)이면 h₂가 h₁을 dominate → h₂가 더 효율적(실제값에 가까울수록 좋음).
  • Relaxed Problem(완화 문제): 제약을 푼 문제의 최적해 비용 = 원문제의 admissible heuristic.
📊 표·그래프 인사이트 (8-puzzle 평균 확장 노드)

🌍 실세계 시나리오 / 완화로 h 발명
8-puzzle에서 "타일이 아무 데나 순간이동 가능"으로 규칙을 풀면 그 최적해=h₁(오배치 수). "타일이 인접칸으로만 이동 가능"으로 풀면 그 최적해=h₂(맨해튼). → 규칙을 느슨히 한 쉬운 문제의 정답이 곧 원문제의 admissible 추정.
⚠️ 함정
admissible은 "과소추정"이 아니라 "과대추정 안 함(≤)". (같은 값도 허용.) Consistency(일관성)는 ⚠️ PDF 미수록 — 시험은 admissible 위주로 충분할 가능성.
📝 시험 포인트 (서술형)"admissible의 정의(부등식·optimistic)와, dominance가 효율에 주는 영향, relaxed problem으로 휴리스틱 만드는 법을 설명하라." → 세 개념을 8-puzzle로 묶어 서술.
🔗 연결선행 C8 / ⚠️ 자료 없음: consistency 상세
C10 · Local Search & Hill-Climbing ⭐高선행: C6▶
🎯 한 줄 직관경로를 기억하지 않고 현재 한 상태만 들고 이웃으로 이동, 목적함수 개선만 추구. ("안개 속 에베레스트 정상 찾기.")
📖 정의 / 언제 쓰나경로가 무의미하고 최종 상태만 중요한 문제(8-queens, n-queens). 장점: 메모리 거의 안 씀, 크거나 무한(연속) 공간에서 합리적 해. Pure optimization: 모든 상태에 목적함수, 목표=max/min 상태, path-cost 틀에 안 맞음.
🔍 작동 원리Hill-Climbing = 값이 증가하는 방향으로 계속 이동, peak에서 종료("greedy local best search"). 즉각 이웃만 보고 lookahead 없음.
🌍 실세계 시나리오
n-queens 자동 배치 — 퀸들을 일단 아무렇게나 놓고, "서로 공격하는 쌍 수(h)"가 줄도록 한 퀸씩 옮긴다. 경로(어떻게 옮겼나)는 아무도 안 궁금하고 최종 충돌 0 배치만 필요 → local search가 적격.
📊 표·그래프 인사이트 (8-queens 정량)
h=서로 공격하는 queen 쌍 수(최소화). 무작위 시작 시 14%만 해결, 86%는 local minimum. 성공 시 평균 4 step, 실패 시 평균 3 step(상태공간 ~17M). Random restart: p=0.14, 99% 성공에 (0.86)ⁿ<0.01 → n≈31회, 최대 step ≈4×31=124.
✏️ 단계별 풀이 예제
8-queens Hill-Climbing + Restart 계산
1. 직관
충돌(공격하는 쌍)을 점수 h로 보고, 그 점수를 깎는 방향으로 퀸을 옮긴다.
2. 셋업
보드에 퀸 8개를 한 열에 하나씩 무작위 배치. h=서로 공격하는 퀸 쌍 수.
3. 전개1·무엇을
각 퀸을 같은 열의 다른 행으로 옮기는 모든 이웃을 평가 → h가 가장 작아지는 이동을 채택.
4. 전개2·왜
h가 더 못 줄면(이웃 모두 ≥ 현재) local minimum — 멈춤. 성공 평균 4 step, 실패 평균 3 step.
5. 결론
단발 성공률 p=0.14(=14%). 86%는 갇힘.
6. 검산·restart
99% 성공: (0.86)ⁿ < 0.01 → n > 30.5 → 약 31회 재시작. 총 비용 ≈ 4×31 = 124 step.
7. 시험 변형
"왜 hill-climbing은 종종 실패하고, 어떻게 구제하나?" → local minimum + multiple random restarts. 이 구조가 C16(min-conflicts)·C19(k-means seed)·C21로 재등장.
⚠️ 함정
local maxima/minima에서 suboptimal로 멈춤. 해법=Multiple Random Restarts. (Day3 "but how many times?" / Day5: greedy local로는 shortest path 못 찾고 끊김, restart하면 state는 찾음.)
📝 시험 포인트 (서술형)"local search의 특징 셋(상태 1개·경로 무시·저메모리)과, hill-climbing이 suboptimal에 빠지는 이유·해법을 설명하라." → local maxima + random restart.

B3. 탐색 — Adversarial (게임 탐색)

C11 · 게임 설정 & Minimax ⭐高선행: C1관문▶
🎯 한 줄 직관적이 내 이익을 깎으려 둔다고 가정하고, 최악의 경우를 최대화(maximin) 하는 수를 둔다.
📖 정의 / 게임 가정 4가지Deterministic, Turn-taking, Zero-sum, Perfect information. 두 플레이어 MAX·MIN, MAX 먼저, 교대.
  • Search vs Games: 게임의 해 = 모든 상대 응수에 대한 전략(strategy)(단일 경로가 아님). 보통 시간제한으로 근사해.
  • 게임 4요소: Initial state, Successor(합법수), Terminal test, Utility function(tic-tac-toe +1/−1/0).
🧮 수식 (Minimax 값)MINIMAX(n) = UTILITY(n) [terminal]; \( \max_{s \in succ} \) MINIMAX(s) [MAX]; \( \min_{s \in succ} \) MINIMAX(s) [MIN].
🌍 실세계 시나리오
체스/틱택토 AI — 내 차례엔 점수를 최대화(MAX), 상대 차례엔 상대가 내 점수를 최소화(MIN)한다고 가정하고 트리를 끝까지(혹은 깊이 d까지) 평가해, 상대가 완벽히 둬도 보장되는 최선의 수를 둔다.
📊 인사이트
트리 크기 O(b^d), b=분기, d=양 플레이어 합산 수. 체스 b~35,d~100 → ≈10¹⁵⁴(탐색 불가). 복잡도 Time O(b^d), Space O(bd)(DFS 방식). MIN이 비최적이면 MAX는 오히려 더 좋아짐(손해 없음). Multiplayer는 minimax 값이 벡터, 비zero-sum이면 동맹(alliance) 유효.
✏️ 단계별 풀이 예제
Minimax 값 backup (잎→뿌리)
1. 직관
잎(끝난 게임)의 점수를 알고, 그 위로 한 칸씩 "내 차례면 최대, 상대 차례면 최소"를 골라 올린다.
2. 셋업
깊이 2 트리: 루트=MAX, 그 아래 MIN 노드들, 그 아래 잎의 utility.
3. 전개1
각 MIN 노드: 자식 잎들 중 최소값을 그 노드 값으로. (상대는 내 점수를 깎으려 함.)
4. 전개2
루트 MAX: 방금 구한 MIN 값들 중 최대값을 선택 → 그 방향이 MAX의 최선수.
5. 결론
루트 값 = 양쪽 최적 플레이 시 MAX가 보장받는 결과.
6. 검산
한 MIN 노드의 자식 하나를 더 나쁘게 바꿔도, MIN이 이미 더 작은 값을 골랐다면 루트 불변.
7. 시험 변형
"이 트리를 α-β로 풀면 어디가 잘리나?" → C12로 연결.
⚠️ 함정
minimax는 "양쪽 모두 최적 플레이" 가정. 상대 실수 시 손해 없음(이득만).
📝 시험 포인트 (서술형)"게임 4가정과 minimax의 재귀 정의를 설명하고, MIN이 실수하면 MAX에게 어떤 일이 생기는지 답하라." → 4가정 + max/min 재귀 + "MAX는 손해 없음(더 좋아짐)".
🔗 연결선행 C1 / 응용 C12(α-β), C13(evaluation fn)
C12 · Alpha-Beta Pruning ⭐高선행: C11▶
🎯 한 줄 직관최종 결정에 영향 없는 가지를 잘라 minimax와 똑같은 결과를 더 빨리.
📖 메커니즘DFS로 단일 경로만 보며 — α=MAX 경로에서 지금까지 찾은 최고값(하한), β=MIN 경로에서 지금까지 찾은 최저값(상한). 노드 값이 조상의 α(MAX)/β(MIN)보다 나빠지는 순간 나머지 가지 prune.
📊 표·그래프 인사이트 (복잡도)
경우복잡도조건
WorstO(b^d)최악 ordering, prune 전혀 안 됨(=minimax)
BestO(b^{d/2})완벽한 ordering(최선수가 항상 leftmost)
실무 평균은 best에 가까움 → 유효 분기수 √b. 체스 b~35 → ~6 → 같은 시간에 약 2배 깊이 탐색.
🌍 실세계 시나리오
Deep Blue(체스) — α-β로 분기를 절반으로 줄여 같은 시간에 더 깊이 읽었다. ordering(좋아 보이는 수를 먼저 탐색)이 prune 효율을 좌우.
✏️ 단계별 풀이 예제
α-β 가지치기 트리 전개
1. 직관
"이 가지를 더 봐도 내 결정이 안 바뀐다"가 보이면 즉시 멈춘다.
2. 셋업
루트 MAX, 두 MIN 자식 L·R. α=−∞, β=+∞로 시작.
3. 전개1
MIN-L의 자식들을 보고 L=3으로 확정 → 루트의 α=3(MAX는 적어도 3 보장).
4. 전개2·왜
MIN-R의 첫 자식이 2로 나옴 → R은 min이라 ≤2가 확정. R(≤2) < α(3)이므로 MAX는 절대 R을 안 고름.
5. 결론
→ R의 나머지 자식들은 볼 필요 없음 = prune. 결과(루트=3)는 minimax와 동일.
6. 검산
prune된 자식이 100이든 −100이든 R≤2는 안 바뀌고 루트는 여전히 3.
7. 시험 변형
"child utility가 모두 같으면 prune이 일어나나?"(Day4 질문) → ordering 효과가 없어 worst에 가까워짐.
⚠️ 함정
prune해도 결과는 minimax와 항상 동일(근사 아님). 효과는 move ordering에 크게 좌우(Day4 교수 질문: ordering 효율화? / 모든 child가 같으면 depth 확장? / alliance 상황 적용?).
📝 시험 포인트 (서술형)"α와 β가 각각 무엇이고, 언제 가지를 자르며, best/worst 복잡도와 ordering의 관계를 설명하라." → α=MAX 하한, β=MIN 상한, "조상 한계를 넘으면 cut", best O(b^{d/2})/worst O(b^d), ordering 의존.
🔗 연결선행 C11 / 같은 결과·더 빠름(근사 아님 강조)
C13 · Utility / Evaluation Function ⭐中선행: C11▶
🎯 한 줄 직관게임을 끝까지 못 읽을 때, 비terminal 보드의 좋음 정도를 점수로 추정.
🧮 수식보통 (내 점수)−(상대 점수). 체스: 선형 가중합 \( Eval(s)=w_1 f_1(s)+\dots+w_n f_n(s) \) (예 w₁=9, f₁=흰퀸수−검은퀸수). 범위 [−∞,+∞] 또는 [−1,+1].
🌍 실세계 시나리오
Deep Blue / Chinook / Othello — Chinook(체커, 1994 인간챔피언 격파), Deep Blue(1997 Kasparov 격파, α-β 기반). Go는 b>300·d>>150로 당시 탐색 불가. Othello eval=흰돌 수−검은돌 수.
📝 시험 포인트 (서술형)"evaluation function이 왜 필요하고 어떤 형태인가?" → 깊이 한계로 terminal까지 못 가니 중간 보드를 선형 가중합으로 평가.
🔗 연결선행 C11 / 함께 C12(깊이 제한 탐색의 잎 평가)

B4. 탐색 — CSP (제약 충족 문제)

C14 · CSP 정식화 & 종류 ⭐高선행: C1관문▶
🎯 한 줄 직관상태를 변수 Xᵢ + 도메인 Dᵢ + 제약(constraints) 으로 정의 → 일반목적 알고리즘 사용 가능.
📖 정의상태=지금까지의 변수 할당, goal test=모든 제약 만족하는 complete & consistent 할당. (black box 상태보다 구조화 → 더 강력.)
🌍 실세계 시나리오
시간표 짜기·스도쿠·지도 색칠 — 호주 Map-Coloring(인접 지역 다른 색; WA,NT,Q,NSW,V,SA,T / D={red,green,blue}). Constraint graph: 노드=변수, 아크=제약(binary CSP). 스도쿠=81변수·도메인1~9·행/열/박스 alldifferent 제약.
📊 인사이트 (종류·복잡도)
  • 변수: Discrete-finite(도메인 d, n변수 → O(dⁿ)), Discrete-infinite(제약언어 필요, 예 StartJob₁+5≤StartJob₃), Continuous(선형제약은 LP로 다항시간).
  • 제약: Unary(SA≠green), Binary(SA≠WA), Higher-order(cryptarithmetic TWO+TWO=FOUR, alldifferent+자리올림).
  • 복잡도: 할당이 commutative("WA=red then NT=green"="그 반대")이므로 한 노드에서 한 변수만 고려 → b=d → dⁿ leaf (순진한 n!·dⁿ가 아님).
⚠️ 함정
leaf가 n!·dⁿ로 보이지만 commutativity로 dⁿ. 이 통찰이 backtracking(C15)의 근거.
📝 시험 포인트 (서술형)"CSP의 세 요소와 goal test(complete&consistent), 그리고 왜 complete 할당 수가 dⁿ인지 설명하라." + constraint graph 정의.
🔗 연결선행 C1·C3(DFS) / 응용 C15·C16
C15 · Backtracking + 효율 향상 휴리스틱 ⭐高선행: C3, C14▶
🎯 한 줄 직관한 노드에 한 변수만 할당하는 DFS(=backtracking). 막히면 되돌아감.
📖 의사코드 골자RECURSIVE-BACKTRACKING(assignment, csp): 완전하면 반환 → 미할당 변수 선택 → 각 값에 consistent하면 추가·재귀, 실패 시 제거 후 다음 값. 모두 실패면 failure. (n-queens n≈25까지.)
🔍 작동 원리 / 효율 향상 4종 (큰 속도 향상)
  • MRV (Minimum Remaining Values / Most Constrained Variable): 남은 합법값이 가장 적은 변수 먼저 → 바로 다음 child 탐색공간 최소화.
  • Most Constraining Variable (Degree heuristic): MRV 동점 타이브레이커 — 남은 변수에 제약을 가장 많이 거는 변수 → grandchild 분기 최소화.
  • LCV (Least Constraining Value): 변수 정한 뒤, 남은 변수의 값을 가장 적게 배제하는 값 선택. (MRV+LCV로 1000-queens 가능.)
  • Forward Checking: 미할당 변수의 남은 합법값 추적, 0이 되면 즉시 종료(조기 실패 감지).
🌍 실세계 시나리오
스도쿠 풀이기 — MRV로 "후보 숫자가 가장 적은 칸"을 먼저 채우고, forward checking으로 한 칸이 채워질 때마다 같은 행/열/박스 후보를 지운다. 후보가 0인 칸이 생기면 즉시 되돌아간다.
✏️ 단계별 풀이 예제
4-Queens를 Forward Checking으로
1. 직관
열마다 퀸 하나(X1~X4, 값=행 1~4). 한 열 정할 때마다 충돌 행을 다른 열 후보에서 지운다.
2. 셋업
도메인 모두 {1,2,3,4}. X1부터 할당, 같은 행·대각선 공격 제약.
3. 전개1
X1=1 시도 → forward checking이 X2,X3,X4에서 충돌 행 제거. 진행하다 X3·X4 후보가 0이 되면 막힘.
4. 전개2·왜
backtrack → X1=2 채택. X2 후보에서 2와 대각 제거 → X2=4 가능.
5. 전개3
X2=4 → forward checking으로 X3,X4 좁힘 → X3=1 가능.
6. 결론
X4 후보에 3만 남음 → 해 X1=2, X2=4, X3=1, X4=3.
7. 검산
네 퀸이 같은 행·대각선에 없음 확인.
8. 시험 변형
"MRV를 쓰면 어느 변수를 먼저 고르나?" + "forward checking이 막힘을 언제 감지하나?(후보 0)".
⚠️ 함정
MRV(변수 선택)와 LCV(값 선택)는 방향이 반대 — 변수는 가장 제약 많은 것, 값은 가장 제약 적은 것.
📝 시험 포인트 (서술형)"backtracking이 무엇이고, MRV·Degree·LCV·Forward Checking이 각각 무엇을 최소화/언제 쓰는지 설명하라." → 한 변수씩 DFS + 네 휴리스틱의 역할. (MRV↔LCV 방향 반대 강조.)
🔗 연결선행 C3·C14 / 응용 C16(완전상태에서 시작하는 반대 접근)
C16 · Local Search for CSP — min-conflicts ⭐⭐ 졸업단골선행: C10, C15최우선▶
🎓 졸업시험 단골 — 최우선 암기. "min-conflicts 절차 + Hill-Climbing vs Simulated Annealing"을 한 문단으로.
🎯 한 줄 직관모든 변수를 일단 다 할당(complete state, 제약 위반 허용)한 뒤, 충돌을 줄이는 방향으로 값을 재배정하는 hill-climbing.
📖 메커니즘
  • 변수 선택 = 무작위로 충돌하는(conflicted) 변수 하나 선택.
  • 값 선택 = min-conflicts 휴리스틱 = 위반 제약 수가 최소가 되는 값. (= h(n)=총 위반 제약 수로 hill-climbing.)
🌍 실세계 시나리오
대규모 시간표/N-queens — 퀸을 다 깔아 놓고(충돌 있어도 OK) 충돌하는 퀸을 골라 그 열에서 공격을 가장 적게 받는 행으로 옮긴다. 백트래킹보다 훨씬 큰 규모에서 빠름.
📊 인사이트
4-queens는 4⁴=256 상태, h=공격 수. 무작위 초기에서 임의 n에 대해 거의 상수 시간에 해결 — n=10,000,000도 높은 확률로 푼다.
✏️ 단계별 풀이 예제 ⭐⭐
min-conflicts로 N-Queens 해결
1. 직관
백트래킹은 빈 보드에서 하나씩 놓지만, min-conflicts는 다 놓고 고친다.
2. 셋업
각 열에 퀸 하나를 무작위 행에 배치(complete state). h=서로 공격하는 쌍 수(>0 가능).
3. 전개1·변수 선택
현재 충돌에 연루된 퀸(conflicted variable) 하나를 무작위로 고른다.
4. 전개2·값 선택
그 퀸을 같은 열의 어느 행으로 옮겨야 공격받는 수가 최소(min-conflicts)가 되는지 계산해 그 행으로 이동.
5. 전개3
h가 0이 될 때까지 3~4 반복. (충돌 0 = 해.)
6. 결론
매우 큰 n(수백만)에서도 거의 상수 시간에 수렴 → CSP local search의 위력.
7. 검산·한계
만약 h가 안 줄고 멈추면 local optimum → 무작위 재시작 또는 SA(확률적 악수 허용)로 탈출.
8. 시험 변형 ⭐⭐
"min-conflicts 절차를 설명하고, hill-climbing과 simulated annealing을 비교하라" — 졸업시험 단골 그대로. 답: 절차 4단계 + HC는 local optimum에 갇힘/SA는 나쁜 수 확률 허용으로 탈출.
⚠️ 함정
hill-climbing이라 local optimum에 갇힐 수 있음 → Simulated Annealing(SA) 은 확률적으로 나쁜 수도 허용해 탈출. (⚠️ SA 온도스케줄·수용확률 수식은 PDF 미수록 — 개념만: "나쁜 수 확률적 허용으로 local optimum 탈출".)
📝 시험 포인트 (⭐⭐ 졸업 단골, 서술형) (1) min-conflicts 절차를 말로: complete state로 시작 → conflicted 변수 선택 → 위반 최소 값으로 재배정 → 반복.
(2) 왜 빠른가: 큰 n에서도 거의 상수 시간(backtracking과 대비).
(3) Hill-Climbing vs Simulated Annealing: HC는 local optimum에 갇힘 / SA는 확률적 악수 허용으로 탈출.
→ 이 세 가지를 묶어 한 문단으로 서술할 수 있게 최우선 암기.
🔗 연결선행 C10(hill-climbing) + C15(backtracking과 대조: complete vs partial) / ⚠️ 자료 없음: SA 상세 수식

B5. 학습 (Learning)

C17 · kNN (k-Nearest Neighbor) Classifier ⭐高선행: 없음(거리 개념)독립▶
🎯 한 줄 직관새 인스턴스를 가장 가까운 k개 이웃의 다수결(majority vote) 로 분류. 모델을 학습하지 않음.
📖 정의instance-based / lazy learning — 분류함수를 국소적으로만 근사, 모든 계산을 질의 시점으로 미룸(pre-training 없음, 데이터만 보유). k=1이면 최근접 1개 클래스.
🧮 거리 수식Euclidean \( d_E(x,y)=\sqrt{\sum_{i=1}^{N}(x_i-y_i)^2} \) / Manhattan \( d_A(x,y)=\sum_{i=1}^{N}|x_i-y_i| \). 인스턴스=n차원 벡터 ⟨a₁,…,aₙ⟩. 정규화 \( x'=\frac{x-\bar{x}}{\sigma(x)} \).
🌍 실세계 시나리오
추천/진단 — "이 사용자와 가장 비슷한 k명이 좋아한 영화"로 추천, "증상이 가장 비슷한 과거 환자 k명의 진단"으로 분류. 미리 모델을 안 만들고 질의 때 비교만 하므로 데이터가 바뀌어도 재학습 불필요(대신 질의가 느림).
✏️ 단계별 풀이 예제 (PlayTennis, 시험 단골)

3. 전개1·정규화
각 열을 \( x'=(x-\bar x)/\sigma \)로 스케일 — 차원 크기 차로 인한 거리 왜곡 제거. (정규화 안 하면 큰 축이 지배해 오분류.)

8. 시험 변형
"정규화를 빼면 결과가 어떻게 달라지나?" / "거리식을 맨해튼으로 바꾸면?"
📊 표·그래프 인사이트
그림 — X차원이 크면 정규화 전엔 오분류(white), 정규화 후 정분류(black). 개선: Distance-Weighted NN(가중치 1/d 또는 1/d²; k=전체면 Shepard's method). Voronoi diagram=1-NN 결정경계. 장단점: Pros=단순·노이즈/복잡 타깃에 효과적·학습 거의 없음; Cons=분류 시간 비용 큼(매 질의마다 전 데이터와 거리계산) → k-d tree로 완화.
⚠️ 함정
"학습"이 아니라 lazy(질의 시 계산). scaling 안 하면 큰 스케일 축이 거리를 지배.
📝 시험 포인트 (서술형)"kNN이 lazy learning인 이유, majority voting, 정규화가 왜 필요한지, k 선택의 영향, Voronoi를 설명하라." → 특히 정규화 필요성(차원 스케일 차이로 거리 왜곡)이 단골.
🔗 연결거리개념 → 응용 C19(k-means)
C18 · Decision Tree ⭐高선행: 없음독립▶
🎯 한 줄 직관내부노드=속성 테스트, 가지=속성값, 잎=분류값 → 데이터를 순도 높은(homogeneous) 그룹으로 재귀 분할.
📖 정의 / 종류큰 이질 집단을 더 작고 동질적인 그룹으로 나누는 split rule 집합. Binary(2분기) / N-way(3분기↑).
🔍 작동 원리 / Best Split = Purity잎이 한 클래스만 = pure. 순도를 가장 많이 증가시키는 split이 best(+ 비슷한 크기 노드 선호). 불순도 측도 4종 중 강의는 Information Gain(entropy) & Gini만.
🧮 수식
  • Entropy: \( E(p)=\sum_j -p_j\log_2 p_j \). 균등분포에서 최대(가장 무질서). (Day6: −log=놀람의 정도, 압축에서 빈출=짧은 코드.)
  • Information Gain: \( Gain(S,A)=E(S)-\sum_{v\in Values(A)}\frac{|S_v|}{|S|}E(S_v) \) = 분할 전후 불순도 차. Gain 최대 속성이 root.
  • Gini: \( Gini=\sum_j p_j^2 \). (root 0.5²+0.5²=0.5=impure, 잎 0.1²+0.9²=0.82=거의 pure.)
🌍 실세계 시나리오
신용 승인/이탈 예측 — "소득→대출이력→나이" 식으로 질문을 분기해 승인/거절을 결정. 규칙이 사람이 읽는 if-then(business rule)로 그대로 매핑돼 해석이 쉽다.

6. 전개4·재귀
pure 잎(Standard)은 고정. 나머지에서 2차 반복 → root=Gender(Gain 0.322), 3차 반복 → Car Ownership로 마무리.
7. 검산·Gini로 교체
Entropy 대신 Gini를 넣어도 동일하게 best split을 고른다(측도만 교체).
8. 시험 변형 ⭐
"Decision Tree에서 가장 중요한 게 뭐냐?" → best split=순도를 가장 높이는(=information gain 최대) 속성 선택. 절차를 말로 설명.
📖 장단점장점=이해 쉬움·business rule 매핑·선험 가정 없음·수치/범주 모두. 단점=타깃이 범주형이어야·단일 타깃·때때로 불안정(local search 유사, Day6 "왜?")·수치속성은 트리 복잡.
⚠️ 함정 (학생 노트 정정)
"Gini는 높을수록 좋다"는 순도 측면(한 클래스만 있으면 Σpⱼ²=1) — 경제학 Gini(불평등)와 혼동 금지. 본 강의 정의에선 분할 후 자식 Gini가 높을수록 pure(좋은 split). (엔트로피와 방향 반대.)
📝 시험 포인트 (⭐ Day7 교수 예시 "Decision Tree에서 가장 중요한 게?", 서술형)답 = best split / purity / information gain. + entropy·gain·gini 수식과 root 선택 절차, 장단점. 증명 X, 개념·절차 O.

C19 · Clustering 개요 & k-means ⭐高선행: C17(거리)▶
🎯 한 줄 직관라벨 없는 데이터를 유사한 것끼리 묶는 비지도학습(cluster 내 유사·간 상이).
📖 정의 / 분류unsupervised의 가장 흔한 형태(IR 문서군집, Google News). 유사도=cosine 또는 Euclidean. Flat(k-means) vs Hierarchical(agglomerative/divisive). 원칙: trivial 클러스터(너무 크거나 작음) 회피.
🧮 수식 / 알고리즘
  • centroid: \( \vec{\mu}(c)=\frac{1}{|c|}\sum_{\vec{x}\in c}\vec{x} \) (무게중심).
  • 의사코드: K개 random seed → {각 doc를 가장 가까운 centroid에 배정 → 각 cluster centroid 재계산} 수렴까지.
  • 종료조건 3종: ①고정 반복수 ②모든 cluster 불변 ③모든 centroid 불변. (②와 ③은 보통 같으나 특수 상황 다름.)
🌍 실세계 시나리오
고객 세분화(customer segmentation) — 구매 패턴 벡터를 k개 그룹으로 묶어 마케팅 타깃을 나눈다. 라벨(정답 세그먼트)이 없으니 비지도, centroid=각 세그먼트의 "평균 고객".
✏️ 단계별 풀이 예제
k-means 반복
1. 직관
"중심을 정하고→가까운 점을 모으고→중심을 다시 계산"을 반복하면 자연히 덩어리로 갈린다.
2. 셋업
K=2, 데이터 {A,B,C,D,E,F}. 초기 seed 무작위로 2개(예 B,E).
3. 전개1·배정
각 점을 더 가까운 seed의 클러스터에 배정 → {A,B,C} / {D,E,F}.
4. 전개2·갱신
각 클러스터의 centroid를 멤버 평균 \( \vec\mu=\frac1{|c|}\sum \vec x \)로 재계산.
5. 전개3·반복
새 centroid로 재배정 → 변화 없으면 종료(종료조건 ②/③).
6. 검산·seed 민감
seed를 D,F로 바꾸면 → {A,B,D,E}/{C,F}로 다른 결과 → local optimum/seed 민감성 입증.
7. 결론
그래서 다중 시도(restart) 또는 먼 벡터 seed 휴리스틱을 쓴다. 복잡도는 O(IKnm) 선형.
8. 시험 변형
"seed가 결과를 바꾸는 이유와 대처법?" / "종료조건 ②와 ③이 다를 수 있는 경우?"
📊 인사이트
시간복잡도 O(IKnm) = 선형(m=차원,n=문서,K=클러스터,I=반복). 수렴: 항상 fixed point 존재, k-means=EM의 특수경우(수렴 증명됨). 초기 seed 민감 → 다른 seed면 다른 결과(예 {A..F}에서 B,E시작→{A,B,C}{D,E,F} vs D,F시작→{A,B,D,E}{C,F}). 해법=먼 벡터 seed 휴리스틱/다중 시도.
⚠️ 함정
local optimum·seed 민감 → 다중 restart. 종료조건 ②≠③(특수 경우).
📝 시험 포인트 (서술형)"centroid 수식과 4단계 절차, 종료조건 3종, O(IKnm) 선형 복잡도, seed 민감성과 EM 수렴을 설명하라."
🔗 연결선행 C17(거리) / 대조 C20(hierarchical) / 연결 C10(seed 민감=hill-climbing 성질)
C20 · Hierarchical Clustering (single/complete/group-average) ⭐高선행: C19▶
🎯 한 줄 직관데이터를 트리(덴드로그램)로 묶는다 — bottom-up(병합) 또는 top-down(분할).
📖 두 방식 / 절차Divisive(Top-Down)=전체를 하나로 보고 재귀 분할(예 k-means). Agglomerative(Bottom-Up)=개별에서 시작해 유사한 것 병합. HAC: ①각 문서를 자기 클러스터로 ②모든 쌍 유사도 계산 ③가장 유사한 쌍 병합 ④클러스터>1이면 ②로.
📊 표·그래프 인사이트 (cluster-cluster 유사도 3종, 시험 핵심)
방식sim 정의성향
Single-linkmax(가장 가까운 쌍)적고 크고 느슨, chaining 효과
Complete-linkmin(가장 먼 쌍)많고 작고 단단, IR 적합·비용 큼
Group-average모든 쌍 평균중간(intermediate)
🌍 실세계 시나리오
문서/뉴스 계층 분류 — 기사를 한 개씩 클러스터로 시작해 가장 비슷한 둘을 계속 합치면, 임계값으로 자르는 위치에 따라 "정치>국내정치>선거"처럼 세분/대분류를 한 트리에서 얻는다.
✏️ 단계별 풀이 예제 (A~F, 6개)
Single-link HAC 병합
1. 직관
가장 닮은 둘을 계속 합쳐 트리를 키운다. "닮음"의 정의(max/min/avg)가 결과 모양을 바꾼다.
2. 셋업
A~F 6개, 15쌍 유사도 표. single-link: 두 클러스터 유사도=멤버 쌍 중 max.
3. 전개1
가장 유사한 쌍 A–F(0.9) 병합 → 클러스터 AF. 갱신: sim(AF,X)=max(sim(A,X),sim(F,X)).
4. 전개2
다음 최댓값으로 AEF(0.8) → ABEF(0.8) → ABEFD(0.6) → ABCDEF(0.5)까지 병합.
5. 전개3·임계값 자르기
0.85에서 자르면 {AF,E,B,D,C}, 0.65 → {AFEB,D,C}, 0.55 → {AFEBD,C}.
6. 대조·complete-link
같은 데이터에 min 기준이면 병합 순서·결과가 달라짐(0.85→{AF,B,E,C,D}, 0.65→{AF,BE,C,D}, 0.35→{AF,BCE,D}). "checked list" 필요.
7. 결론
single-link는 chaining으로 크고 느슨, complete-link는 작고 단단.
8. 시험 변형 ⭐
"single/complete의 sim 정의 차이와 성향(chaining vs tight)을 덴드로그램으로 설명하라."
⚠️ 함정 (학생 노트 정정)
single-link=max sim(가장 가까운 쌍), complete-link=min sim(가장 먼 쌍) — "Best/Worst"보다 max/min 정의로 외울 것. single-link는 chaining으로 불공정 성장.
📝 시험 포인트 (⭐ 서술형)"single vs complete vs group-average의 sim 정의(max/min/avg)와 성향(chaining vs tight), agglomerative 절차, 덴드로그램을 임계값으로 자르는 법을 설명하라."

C21 · Reinforcement Learning & Q-Learning ⭐高선행: C10⚠️ 기말 범위 미확인▶
⚠️ Notion Day 노트에 Q-learning 강조/시험가이드 없음(PDF 단독 근거). 기말 범위 포함은 진도상 추정이나 명시 확인 불가.
🎯 한 줄 직관에이전트가 시행착오로 reward를 받아 점차 최적 행동을 강화. Q값으로 "각 상태에서 어떤 행동이 유리한지"를 테이블로 학습.
📖 정의ML 3분류 = Supervised/Unsupervised/Reinforcement(행동에 reward로 강화). RL 특징: delayed reward(지금 행동의 적절성을 당장 모름), life-long learning. 구조: Agent ↔ Environment (state,reward / action). Q-Learning=RL 중 최다 사용, reward를 전파(propagate).
🧮 갱신 수식 (핵심)\[ Q(s,a) = r(s,a) + \gamma \max_{a'} Q(s',a') \]
r=즉각 reward, γ=전파 계수[0,1](delayed reward 할인), s'=a 후 새 상태, a'=s'의 가능 action. 최적 정책 \( \pi(s)=\arg\max_a Q(s,a) \). 의미: Q(s,a)= 즉각 reward + (a로 간 s'에서 받을 미래 최대 Q)의 할인합.
🌍 실세계 시나리오
게임 에이전트/미로 탈출 로봇 — 로봇이 미로에서 무작위로 움직이며 출구에서만 reward 100을 받는다. 여러 번 시도하면 출구에 가까운 칸일수록 Q가 커져(100→50→25…) Q 그래디언트가 최단 경로를 가리킨다.
✏️ 단계별 풀이 예제 (3×2 그리드)
Q-learning 업데이트 — 3×2 그리드
1. 직관
출구에서만 reward를 받지만, 갱신식이 그 보상을 출구→입구 방향으로 한 칸씩 뒤로 퍼뜨린다.
2. 셋업
S1~S6, S6=END, γ=0.5, 목표 reward=100. 모든 Q=0으로 시작.
3. 전개1
S3→S6(a36) 도달 → \( Q(S3,a36)=r=100 \).
4. 전개2·전파
다음 게임에서 \( Q(S2,a23)=0+0.5\cdot\max Q(S3,*)=0.5\cdot100= \) 50.
5. 전개3
\( Q(S1,a12)=0+0.5\cdot\max Q(S2,*)=0.5\cdot50= \) 25. → 보상이 목표에서 뒤로 전파.
6. 결론·수렴 table
S3,a36=100 / S2,a23=50,S5,a56=100 / S1,a12=25,S5,a54=25 / S2,a21=12.5… 목표에 가까울수록 큼.
7. 검산
γ=0.5라 한 칸 멀어질 때마다 절반 감쇠(100→50→25→12.5) → Q 그래디언트 = 최단 경로.
8. 시험 변형
"γ를 0.9로 바꾸면 전파 값이 어떻게 변하나?" / "Q값에서 정책은 어떻게 뽑나?(argmax)".
📊 그래프 인사이트
목표(S6)에 가까울수록 Q 큼(100→50→25→12.5, γ=0.5씩 감쇠) → Q값의 공간 그래디언트가 최적 경로.
⚠️ 함정
Q는 "즉각 reward"가 아니라 "즉각 + 미래 최대 Q의 할인합". 학습 중엔 임의(탐험) action도 선택.
📝 시험 포인트 (서술형)"Q(s,a) 갱신식의 각 기호(r, γ, max Q(s',a')) 와, reward가 목표에서 뒤로 전파(propagation) 되는 개념, 수렴 후 정책 argmax, γ의 역할을 설명하라."

C22 · ANN — Perceptron & XOR 한계 ⭐高선행: C18⚠️ 기말 범위 미확인▶
⚠️ Notion Day 노트 없음(PDF 단독). 기말 범위 명시 미확인.
🎯 한 줄 직관뉴런(Dendrites·Nucleus·Axon)을 모방한 단위 perceptron. 입력·가중치 곱의 합이 임계치를 넘으면 발화.
📖 구조 / 학습입력 pᵢ×가중치 wᵢ의 합 Σ → activation function fₙ → 출력 y. activation 종류: hard limiter, linear, sigmoid. 뇌(10¹⁰뉴런·병렬·비선형) vs 컴퓨터(빠르나 순차·선형).
🧮 학습 = Widrow-Hoff (delta) rule출력 y(t)와 목표 d(t)의 차이를 최소화. \[ W_i(t+1) = W_i(t) + \alpha[d(t)-y(t)]\,X_i(t) \] (α=학습률). 차이 없으면 가중치 불변, 있으면 줄이는 방향.
🌍 실세계 시나리오
스팸 필터/단순 분류기 — 단어 빈도(입력)에 가중치를 곱해 합이 임계치를 넘으면 스팸. 단, 선형으로 가를 수 있는 문제만 가능.
🔍 작동 원리 / AND 가능, XOR 불가AND는 단일 perceptron 가능(W0,W1=0.3~0.4, θ=0.5). XOR은 만족하는 W0,W1이 없음 = linearly non-separable → 단일 perceptron 불가. 해결: multi-layer(2~3층) → 3층으로 어떤 문제도 근사 가능.
⚠️ 함정
단일 perceptron의 한계는 선형 분리 가능 문제로 제한. XOR이 대표 반례.
📝 시험 포인트 (⭐ 서술형)"perceptron 구조와, XOR이 안 되는 이유(linearly non-separable) 및 multi-layer 해결, delta rule을 설명하라."

C23 · Backpropagation Neural Network ⭐高선행: C22⚠️ 기말 범위 미확인▶
⚠️ Notion Day 노트 없음(PDF 단독). 기말 범위 명시 미확인.
🎯 한 줄 직관input–hidden–output 다층망에서 출력 오차를 gradient descent로 뒤로 전파해 가중치를 갱신 → 단층의 선형분리 한계 극복, 연속함수 근사.
📖 정의input·output 사이 1개↑ hidden layer를 갖는 단방향 신경망. 80년대 중반 Error Backpropagation = generalized delta rule. 학습=목표 d와 출력 o의 오차제곱합 최소화. Gradient Descent: 오차 기울기 반대방향으로 가중치 이동. η(학습률) 너무 작으면 정체, 너무 크면 진동(0.005 vs 0.05 곡선).
🧮 수식
  • Forward: hidden \( Input_{pj}=\sum X_{pi}W_{ij}-\theta_j \), \( O_{pj}=f(Input_{pj}) \) ; output 동형.
  • Error: \( E_p=\frac{1}{2}\sum_k (d_{pk}-O_{pk})^2 \). Sigmoid 도함수 \( \frac{\partial y}{\partial x}=y(1-y) \).
  • Output δ: \( \delta_{pk}=(d_{pk}-O_{pk})\,O_{pk}(1-O_{pk}) \).
  • Hidden δ: \( \delta_{pj}=\left(\sum_k \delta_{pk}W_{jk}\right)O_{pj}(1-O_{pj}) \) — 출력층 δ를 가중합해 뒤로 전파.
  • 갱신: Hidden→Output \( W_{jk} \mathrel{+}= \eta\,\delta_{pk}\,O_{pj} \) ; Input→Hidden \( W_{ij} \mathrel{+}= \eta\,\delta_{pj}\,X_{pi} \).
🌍 실세계 시나리오
함수 근사/패턴 인식 — y=0.5(cos8x+sin4x−x+0.8)를 1-6-1 망으로 1,000→10,000→20,000회 학습하면 점점 정확히 근사. 손글씨·음성처럼 선형으로 못 가르는 패턴을 다층+역전파로 학습.
🔍 작동 원리 (PDF 12단계 요약)①가중치·θ 초기화 ②입력·목표 제시 ③④hidden 입출력 ⑤⑥output 입출력 ⑦gradient 계산 ⑧Hidden→Output 갱신 ⑨Input→Hidden 갱신 ⑩학습쌍 반복 ⑫출력층 오차합 E가 허용치↓ 또는 최대반복 초과면 종료.
⚠️ 함정
hidden δ는 output δ의 가중합 × sigmoid'. 상위→하위 역전파는 생물학적으로 비현실적(실제 뉴런은 상위 목표 모름)이나 가장 널리 쓰임. Day7 시험형식상 유도 손계산은 안 나옴 → 개념·절차 서술이면 충분.

서술형Q1. 정보 없는 탐색 5종(BFS·DFS·UCS·Depth-Limited·IDS) 중 메모리는 선형이면서 완전하고 최적인 전략은 무엇이며 그 이유는? (출처: Notion Day1 / C5b)
✅ 모범 답변IDS(Iterative Deepening Search). DFS는 깊이를 따라가므로 공간이 O(bd) 선형이고, IDS는 depth-limit L을 1씩 키우며 DFS를 반복하므로 BFS의 완전성·최적성(얕은 해 먼저)을 함께 얻는다. 반복 탐색처럼 보이지만 지수 트리에서 노드 수는 가장 깊은 레벨이 지배하므로 재방문 비용은 BFS 대비 약 11%(≈b/(b−1))에 불과하다. 반면 BFS·UCS는 최적이나 지수 공간, DFS는 선형 공간이나 불완전·비최적.
🎯 부분점수 포인트"IDS" 지목(+), 선형 공간 O(bd) 언급(+), 완전·최적 확보 이유(L 증가 반복)(+), 11% 낭비/마지막 레벨 지배 논리(+가산).
연결 노드: C4, C5b
서술형Q2. A*의 평가함수 f(n)=g(n)+h(n)에서 각 항의 의미를 쓰고, A*가 최적이 되는 조건과 그 직관을 설명하라. (출처: Notion Day2 / C8·C9)
✅ 모범 답변g(n)=시작~n까지의 실제 누적비용, h(n)=n~목표까지의 추정 잔여비용. 둘을 더해 "지나온 비용+앞으로의 추정"이 최소인 노드를 확장 → Greedy(h만)의 함정과 UCS(g만)의 느림을 모두 피한다. 최적 조건은 h가 admissible(모든 n에서 h(n)≤h*(n), 즉 절대 과대평가하지 않음=optimistic)일 때(TREE-SEARCH). 과대평가하면 더 싼 경로를 너무 비싸 보이게 해 건너뛸 수 있어 최적성이 깨진다. 또 엣지 비용 ≥ 0이어야 한다.
🎯 부분점수 포인트g·h 정의 각각(+), admissible 부등식 h≤h*(+), "과대평가 안 함=optimistic"(+), 음수 비용 불가(+가산), optimally efficient 언급(+가산).
연결 노드: C8, C9
서술형Q3. Minimax에서 MIN이 최적이 아닌(실수하는) 수를 두면 MAX의 결과는 어떻게 되는가? 그 이유를 게임 가정과 함께 설명하라. (출처: Notion Day4 / C11)
✅ 모범 답변MAX는 손해 보지 않고 오히려 더 좋아질 수 있다. Minimax는 양쪽 모두 최적(infallible) 플레이를 가정하고 MAX의 worst-case를 최대화하도록 수를 고른다 — 즉 MAX의 minimax 값은 "상대가 완벽히 둬도 보장되는 하한"이다. 따라서 MIN이 그 최선의 방어를 못 하면 실제 결과는 그 보장값 이상으로만 갈 수 있다. (게임 4가정: Deterministic, Turn-taking, Zero-sum, Perfect information.)
🎯 부분점수 포인트"손해 없음/더 좋아짐"(+), worst-case 최대화=보장 하한 논리(+), 4가정 언급(+가산).
연결 노드: C11, C12
⭐⭐ 졸업단골Q4. CSP의 Local Search(min-conflicts) 절차를 설명하고, Hill-Climbing과 Simulated Annealing을 비교하라. (출처: Notion Day5 — "졸업 시험 단골!" / C16)
✅ 모범 답변min-conflicts 절차 — ① 모든 변수를 일단 다 할당해 complete state로 시작(제약 위반 허용), ② 충돌에 연루된(conflicted) 변수를 하나 무작위로 선택, ③ 그 변수를 위반 제약 수가 최소가 되는 값(min-conflicts 휴리스틱) 으로 재배정, ④ 충돌(h=총 위반 수)이 0이 될 때까지 반복. 이는 h=위반 제약 수에 대한 hill-climbing이다. 매우 큰 n(수백만 queens)에서도 거의 상수 시간에 수렴해 backtracking보다 강력하다. 비교 — Hill-Climbing은 더 나은 이웃이 없으면 멈춰 local optimum에 갇힌다. Simulated Annealing은 확률적으로 나쁜 수도 허용해(초기엔 높은 확률, 점차 감소) local optimum을 탈출한다.
🎯 부분점수 포인트complete state 시작(+), conflicted 변수 선택(+), min-conflicts(위반 최소 값)(+), hill-climbing임 명시(+), 큰 n 상수시간(+가산), HC=local optimum 갇힘 / SA=나쁜 수 확률 허용 탈출 비교(+필수). 졸업단골 — 절차+비교를 한 문단으로.
연결 노드: C16, C10, C15
서술형Q5. kNN에서 정규화(scaling) 가 왜 필요한가? PlayTennis 같은 예로 설명하라. (출처: Notion Day6 / C17)
✅ 모범 답변kNN은 거리(Euclidean/Manhattan)로 최근접 이웃을 찾는데, 차원마다 값의 스케일이 다르면 스케일이 큰 축이 거리를 지배해 작은 축의 정보가 묻히고 오분류가 생긴다. 그래서 각 특성을 \( x'=(x-\bar{x})/\sigma \)로 표준화해 축들을 동등하게 만든다(그림: 정규화 전 white로 오분류 → 정규화 후 black로 정분류). kNN은 lazy learning이라 미리 모델을 만들지 않고 질의 시점에 전 데이터와 거리 계산 후 k개 이웃의 다수결로 라벨을 정한다.
🎯 부분점수 포인트스케일 큰 축이 거리 지배→오분류(+필수), 표준화 식 또는 개념(+), lazy learning·majority voting 언급(+가산).
연결 노드: C17
서술형Q6. "Decision Tree에서 가장 중요한 것은 무엇인가?" (Day7 교수 예시) — 답하고 root 속성 선택 절차를 설명하라. (출처: Notion Day6·Day7 / C18)
✅ 모범 답변가장 중요한 것은 best split = 순도(purity)를 가장 많이 높이는 분할이며, 이를 정량화하는 것이 Information Gain이다. 각 속성으로 나눴을 때 분할 전후 불순도 차 \( Gain(S,A)=E(S)-\sum_v \frac{|S_v|}{|S|}E(S_v) \)를 계산해 Gain이 최대인 속성을 root로 삼고, 순수(pure)해진 잎은 고정한 뒤 나머지에서 재귀한다(예: Transportation에서 Travel Cost가 Gain 1.210으로 root). Entropy 대신 Gini(\( \sum_j p_j^2 \))를 써도 동일하게 best split을 고른다.
🎯 부분점수 포인트best split/purity가 핵심(+필수), information gain으로 측정(+), root=Gain 최대(+), 재귀·pure 잎 고정(+가산), Gini 대체 가능(+가산).
연결 노드: C18
서술형Q7. k-means의 4단계 절차와 시간복잡도, 초기 seed 민감성을 설명하라. (출처: Notion Day7 / C19)
✅ 모범 답변① K개 random seed(centroid) 선택 → ② 각 점을 가장 가까운 centroid의 클러스터에 배정 → ③ 각 클러스터의 centroid를 멤버 평균 \( \vec\mu=\frac{1}{|c|}\sum \vec x \)로 재계산 → ④ 변화 없을 때까지 ②③ 반복(종료조건: 고정 반복수 / 클러스터 불변 / centroid 불변). 시간복잡도는 O(IKnm)로 선형(I=반복, K=클러스터, n=점, m=차원). seed 민감성: 초기 seed에 따라 결과가 달라져 local optimum에 빠질 수 있어(예 {A..F}에서 B,E 시작 vs D,F 시작이 다른 군집) 다중 restart 또는 먼 벡터 seed 휴리스틱으로 완화한다(k-means=EM의 특수경우, fixed point 수렴).

서술형Q8. 계층군집에서 single-link / complete-link / group-average의 cluster 유사도 정의와 성향 차이를 설명하라. (출처: Notion Day8 / C20)
✅ 모범 답변Single-link = 두 클러스터 멤버 쌍 중 가장 가까운(max similarity) 쌍으로 유사도 정의 → 적고 크고 느슨한 클러스터, chaining 효과. Complete-link = 가장 먼(min similarity) 쌍 기준 → 많고 작고 단단한 클러스터, IR에 더 적합하나 비용 큼. Group-average = 모든 멤버 쌍 유사도의 평균 → 둘의 중간. (Agglomerative HAC: 각 점을 클러스터로 시작 → 가장 유사한 쌍을 반복 병합 → 덴드로그램을 임계값에서 잘라 군집 수 결정.) ⚠️ 라벨이 헷갈리므로 single=max, complete=min으로 외울 것.
🎯 부분점수 포인트single=max/가장 가까운 쌍(+), complete=min/가장 먼 쌍(+), group-avg=평균(+), chaining vs tight 성향(+가산), 덴드로그램 임계값 자르기(+가산).
연결 노드: C20, C19
서술형Q9. Q-learning의 갱신식 \( Q(s,a)=r(s,a)+\gamma\max_{a'}Q(s',a') \)에서 각 기호의 의미와, reward가 목표에서 뒤로 전파(propagation) 되는 과정을 설명하라. (출처: 7번 PDF / C21 · ⚠️ 기말 범위 미확인)
✅ 모범 답변r(s,a)=상태 s에서 행동 a로 받는 즉각 reward, γ∈[0,1]=미래 보상 할인(전파) 계수, s'=a 후의 새 상태, \( \max_{a'}Q(s',a') \)=s'에서 가능한 최선 행동의 가치. 즉 Q(s,a)=즉각 reward + 미래 최대 Q의 할인합. 학습은 모든 Q=0에서 시작해 시행착오로 행동하며 이 식으로 갱신한다. 목표에 닿은 상태가 먼저 큰 Q를 얻고(예 Q=100), 그 인접 상태가 다음 게임에서 γ·100=50, 그 다음이 25…로 보상이 목표→출발 방향으로 한 칸씩 뒤로 전파된다. 수렴 후 정책은 \( \pi(s)=\arg\max_a Q(s,a) \).
🎯 부분점수 포인트r·γ·max Q(s',a') 각 의미(+), "즉각+미래 최대 Q 할인합"(+), 100→50→25 역전파 예(+필수), argmax 정책(+가산).
연결 노드: C21
서술형Q10. 단일 perceptron으로 XOR을 풀 수 없는 이유는 무엇이며, 어떻게 해결하는가? (출처: 8번 PDF / C22·C23 · ⚠️ 기말 범위 미확인)
✅ 모범 답변단일 perceptron은 입력의 가중합이 임계치를 넘는지로 분류하므로 하나의 직선(초평면)으로 가를 수 있는 문제(linearly separable) 만 풀 수 있다. AND는 가능하지만 XOR은 (0,0)·(1,1)→0과 (0,1)·(1,0)→1을 한 직선으로 나눌 수 없어 = linearly non-separable → 만족하는 가중치 W0,W1이 존재하지 않는다. 해결은 multi-layer(2~3층): hidden layer를 두면 비선형 결정경계를 만들 수 있고, 3층 perceptron으로 어떤 문제도 근사할 수 있다. 다층망의 학습은 backpropagation(출력 오차를 gradient descent로 뒤로 전파해 가중치 갱신)으로 한다. (Day7 형식상 유도 손계산은 안 나오고 개념·절차 서술이면 충분.)
🎯 부분점수 포인트linearly non-separable이 이유(+필수), AND 가능/XOR 불가 대비(+), multi-layer/hidden layer 해결(+), 3층 근사·backprop 연결(+가산).
