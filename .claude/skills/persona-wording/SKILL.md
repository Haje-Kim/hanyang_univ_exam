# Persona Wording — haje 어투 재작성

학습 허브 문장을 haje 페르소나의 선호 어투로 다듬는다. 참조 원장은 `D:\user\Documents\hj_works\11_wiki\haje\` vault. 단, `voice/tone.md`는 현재 placeholder이므로 **presentation-style DNA를 산문으로 변환**해 기준으로 삼고, 동시에 tone.md를 정식 작성한다.

## 왜 이렇게 하나

- 발표 덱 DNA는 "전보체(조사 생략)"라 슬라이드엔 맞지만 학습자료엔 그대로 못 쓴다 → **간결체로 한 단계 풀어** 적용한다.
- voice/tone.md가 비어 있으니, 이번에 도출한 규칙을 vault에 남겨 **다음부터 재사용**한다(원장화).

## 재작성 규칙 (presentation-style DNA → 산문)

1. **제목: 한글 명사구, 짧고 굵게.** 동사·서술형 회피. "~하는 방법" → "~의 원리". 필요 시 영문 키커 + 한글 제목 페어링.
2. **본문: 간결체.** 완결 문장을 쓰되 군더더기·중복·번역투 제거. 병렬 항목은 ` · `와 `/`로 압축.
3. **결론부터.** 정의·핵심 먼저, 부연 뒤에. 한 문단 한 메시지.
4. **숫자·고유명사 그대로.** "약 88.8% 정도" → "88.8%". 꾸밈말 제거.
5. **강조는 단호하게.** 주의·필수·함정은 흐리지 않고 명확히("⚠ ~ 필수", "~ 금지").
6. **번역투·LLM투 제거.** 아래 표 참조.

### 걷어낼 표현 (do NOT)

| 어색/LLM투 | 고쳐 쓰기 |
|---|---|
| ~할 수 있습니다만 / ~인 것으로 보입니다 | ~한다 / ~이다 (단정) |
| 다양한, 효과적으로, 매우, 굉장히 | 삭제하거나 구체 수치·근거로 |
| ~를 통해 ~를 진행합니다 | ~로 ~한다 |
| 불필요한 피동(~되어집니다)·이중부정 | 능동·긍정 |
| ~라고 할 수 있다 | ~다 |
| 장황한 도입("먼저 살펴보면, 우리는…") | 바로 본론 |

## 안전 규칙 (의미 보존)

- **워딩만 바꾼다.** 정의·수식·시험 정답·기술적 사실은 불변. 애매하면 원문 의미 유지.
- **수술적 변경.** 이미 자연스러운 문장은 두고, 바꾼 문장은 "어색함 제거" 근거가 있어야 한다.
- **건드리지 말 것:** KaTeX 수식(`$...$`, `\(...\)`), 코드, `id`/`class`명, 앵커. 텍스트 노드만 수정.
- **용어 일관성.** 같은 개념 = 같은 용어. 과목 간 표기 통일.
- **대량 치환 주의.** replace_all은 의미 오변경 위험 → 문맥별 수정 우선.
