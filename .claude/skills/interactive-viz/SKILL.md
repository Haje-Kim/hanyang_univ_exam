# Interactive Viz — 인터랙티브 학습 요소 구현

정적 표·트리맵·"버튼 누르면 div 토글"은 인터랙티브가 아니다. **트리를 그리거나 텍스트를 바꾸는 것**도 그 자체로는 시각화가 아니다. 핵심은 **추상 개념을 움직이는 시각으로 보여 "변화"를 눈으로 따라가게** 하는 것이다(3Blue1Brown DNA). 이 스킬은 그 애니메이션 씬을 오프라인 우선으로 구현하는 방법을 담는다.

> **원장 로드:** `D:\user\Documents\hj_works\11_wiki\haje\working-style\explanation-style.md`의 3B1B 항목(시각 우선·모핑·점진 전개·일관 색)이 이 스킬의 시각 원칙이다.

## 왜 이렇게 하나

- **변화는 모핑으로 보여야 이해된다.** 상태 A→B를 점프컷으로 바꾸면 무엇이 변했는지 모른다. 부드러운 전환으로 보여야 학습자가 따라간다.
- **오프라인 우선이 원칙.** 그래서 애니메이션은 **CDN 라이브러리가 아니라 자체 경량 엔진**(requestAnimationFrame + easing, ~150줄 vanilla)으로 만든다. CDN 의존 0 → 오프라인 100% 보장. 무거운 표현이 꼭 필요할 때만 CDN(p5.js 등)을 얹고 정적 폴백을 둔다.
- **개념 ↔ 시각 1:1.** 모든 시각 요소에 "이게 무엇인지" 라벨. 화려함이 아니라 이해 전달이 목적.

## 패턴 — 언제 무엇을

|---|---|---|

| **D3 탐색 트리** | 과목 전체 구조 **내비게이션**(트리맵 대체). 트리는 "시각화"가 아니라 길찾기 도구로 한정 | `references/d3-collapsible-tree.md` |

concept-explainer가 본문에 남긴 `<!-- VIZ: ... | type: step|morph|structure -->` 마커를 입력으로, 그 자리에 해당 패턴을 넣는다.

## 3B1B 애니메이션 씬 — 핵심 (상세는 reference)

- **자체 경량 엔진**(`sceneEngine`): `play(from, to, {duration, easing, onFrame})`로 값/좌표를 보간, requestAnimationFrame으로 그린다. Canvas 또는 SVG. CDN 0.
- **씬 = 단계 시퀀스.** 각 단계는 "상태"이고, 단계 사이는 보간으로 모핑. 재생/일시정지/단계 스크럽(앞뒤) 컨트롤 제공.
- **구체 먼저·점진 등장.** 처음부터 다 그리지 않는다. 요소를 하나씩 페이드/슬라이드로 등장.
- **일관 색.** 한 색 = 한 의미(예: 비교 중=accent, 확정=green, 후보=muted)를 씬 전체에서 유지. `var(--accent)` 등 토큰 사용.
- **라벨·캡션.** 현재 단계가 무엇을 하는지 텍스트 캡션을 씬 아래 동기 갱신.

## 공통 규칙 (전 패턴)

1. **오프라인 우선.** 3B1B 씬·시뮬레이션은 자체 엔진(CDN 0)이 기본. D3 트리만 CDN+폴백 쌍(`typeof d3 !== 'undefined'` 확인, 실패 시 정적 트리맵). 빈 화면 금지.
2. **개념 라벨 필수.** 모든 시각 요소에 라벨, 시뮬/씬엔 현재 단계 캡션.
3. **토큰 재사용.** 색은 `var(--accent)` 등 기존 변수. 다크모드 전환 시 Canvas/SVG 색도 따라가게(테마 변경 시 재렌더).
4. **반응형.** SVG는 `viewBox`+`width:100%`, Canvas는 devicePixelRatio 스케일+리사이즈 핸들러. 모바일 컨트롤 세로 스택.
5. **접근성.** 자동재생 금지(사용자가 play). `prefers-reduced-motion`이면 모핑 대신 단계 점프 + 정적 표시. 키보드로 단계 이동.
6. **기존 동작 보존.** onclick·스크롤스파이·앵커를 깨지 않게 통합. id 변경 시 참조처 갱신.
7. **점진 검증.** 한 씬을 끝낼 때마다 harness-qa에 오프라인 로드·렌더·콘솔 에러·reduced-motion 폴백을 확인 요청.

## 안티패턴 (하지 말 것)

- 트리를 그리거나 div를 토글하고 "인터랙티브 시각화"라 부르기.
- 상태 변화를 점프컷으로 바꾸기(모핑 없이) → 무엇이 변했는지 안 보임.
- 자동재생·과한 모션 → reduced-motion 무시, 학습 방해.
- 씬에 라벨·캡션 없이 애니메이션만 재생 → 학습 효과 0.
- 3B1B 씬을 CDN 라이브러리에 의존시켜 오프라인에서 죽이기.
