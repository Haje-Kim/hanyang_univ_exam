# Interactive Viz — 인터랙티브 학습 요소 구현

정적 표·트리맵·"버튼 누르면 div 토글"은 인터랙티브가 아니다. 학습자가 **파라미터를 바꾸면 결과가 반응**하거나 **단계를 직접 전진/후진하며 상태를 관찰**할 수 있어야 한다. 이 스킬은 그 세 가지 패턴을 오프라인 우선으로 구현하는 방법을 담는다.

## 왜 이렇게 하나

- **오프라인 우선이 원칙**이라 외부 라이브러리는 항상 깨질 수 있다. 그래서 CDN 로드 + 실패 폴백을 항상 쌍으로 둔다 — 라이브러리가 없어도 학습 핵심은 보여야 한다.
- **개념 이해가 목적**이다. 화려한 애니메이션이 아니라, 시각 요소 하나하나가 어떤 개념·변수에 대응하는지 라벨로 드러나야 학습에 쓸모가 있다.
- **기존 자산 재사용.** 각 HTML의 `:root` CSS 토큰·다크모드·KaTeX 렌더를 그대로 쓴다. 새 색·폰트를 들이면 일관성이 깨진다.

## 세 가지 패턴 — 언제 무엇을

| 패턴 | 쓸 곳 | 참조 |
|---|---|---|
| **D3 탐색 트리** | 과목 전체 구조(기존 트리맵·로드맵 대체). 접기/펼치기·줌·노드 클릭 드릴다운 | `references/d3-collapsible-tree.md` |
| **개념 시뮬레이션** | 단계가 있는 개념(정렬·DP·탐색·CSP·GNN 메시지패싱·ROC 임계값) | `references/simulations.md` |
| **인터랙티브 논문 포스터** | 의생명 핵심 논문(DNABERT·Deep Learning·GAT) 모듈별 심화 | `references/paper-poster.md` |

각 패턴의 완성 코드·폴백 처리·접근성 체크리스트는 해당 reference를 로드해 따른다.

## 공통 규칙 (3패턴 전부)

1. **CDN + 폴백 쌍.** 라이브러리 로드 성공 여부를 `typeof d3 !== 'undefined'`로 확인하고, 실패 시 기존 정적 표현(트리맵 div·텍스트 표)을 노출한다. 빈 화면 금지.
2. **개념 라벨 필수.** 모든 시각 요소에 "이게 무엇인지" 라벨을 붙인다. 시뮬레이션엔 현재 단계 설명 텍스트를 함께 갱신한다.
3. **토큰 재사용.** 색은 `var(--accent)` 등 기존 변수로. 다크모드 전환 시 SVG/캔버스 색도 따라가게 한다.
4. **반응형.** SVG는 `viewBox` + `width:100%`. 모바일에서 트리는 세로 스크롤·핀치줌 허용, 시뮬레이션 컨트롤은 세로 스택.
5. **기존 동작 보존.** onclick 핸들러·스크롤스파이·앵커 이동을 깨지 않게 통합한다. id를 바꿔야 하면 참조처도 함께 갱신.
6. **점진 검증.** 한 모듈을 끝낼 때마다 harness-qa에 오프라인 로드·렌더·콘솔 에러 확인을 요청한다.

## 작업 순서

1. content-architect의 목차 트리(`_workspace/{과목}_toc.md`)를 받아 트리 데이터(JSON 계층)로 변환한다.
2. 대상 HTML에서 교체할 정적 요소(`.treemap`, 로드맵 nav) 위치를 찾는다.
3. 해당 패턴 reference를 로드해 코드를 삽입하고, 폴백을 연결한다.
4. KaTeX·다크모드·반응형이 함께 동작하는지 자가 확인 후 QA에 넘긴다.

## 안티패턴 (하지 말 것)

- 라이브러리 없이 div를 토글하고 "인터랙티브"라 부르기.
- 폴백 없이 CDN에만 의존 → 오프라인에서 빈 화면.
- 시뮬레이션에 라벨·단계 설명 없이 애니메이션만 재생 → 학습 효과 0.
- 기존 색·폰트를 무시하고 D3 기본 스타일을 그대로 노출.
