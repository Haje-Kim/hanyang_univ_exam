# Content Architecture — 목차 정리 + 선행지식 파트

학습자가 본 개념에 들어가기 전 **토대(선행지식)**를 갖추게 하고, 과목 전체를 **가독성 있는 목차**로 정렬한다. 이 스킬의 산출물(목차 트리)은 interactive-viz의 탐색 트리 데이터 소스가 된다.

## 왜 이렇게 하나

- 목차가 산만하면 탐색 트리도 산만해진다. 트리 시각화 전에 **정보 구조부터** 바로잡아야 한다.
- 학습자가 전문 용어를 모른 채 본문에 들어가면 이해가 막힌다. **선행지식을 앞에 모으면** 본문은 개념 설명에 집중할 수 있다.

## A. 목차 재점검·정리

1. **현재 구조 추출.** 대상 HTML의 섹션/노드 id·제목을 모두 모아 현 목차를 만든다.
2. **MECE 점검.** 같은 개념이 흩어졌으면 모으고, 겹치면 합치고, 빠진 연결고리를 채운다.
3. **위상 정렬.** 선행지식 → 핵심 → 심화. 뒤 개념이 앞에 의존하면 순서를 맞춘다.
4. **명칭 정리.** 제목은 한글 명사구로 짧게(워딩 세부는 persona-wording이 담당, 여기선 구조만).
5. **트리 데이터 산출.** 정리 결과를 `_workspace/{과목}_toc.md`에 계층(들여쓰기 + id)으로 저장. 각 노드 id는 본문 섹션 앵커와 1:1.

**id 안전 규칙:** 기존 id를 바꾸면 로드맵 링크·스크롤스파이가 깨진다. 가능하면 기존 id를 유지하고, 새 섹션에만 새 id를 부여한다. 불가피하게 바꾸면 참조처(onclick·href·스크롤스파이 목록)를 함께 갱신하고 QA에 알린다.

## B. 선행지식(Prerequisites) 파트 신설

본문 맨 앞(개요 다음, 첫 핵심 개념 전)에 "선행지식" 섹션을 만든다. 과목별 핵심 용어:

| 과목 | 선행지식 용어(예) |
|---|---|
| 인공지능개론 | 상태공간·탐색·휴리스틱·지역최적/전역최적·확률·조건부확률·복잡도 |
| 컴퓨터 알고리즘 | 점근표기(O/Θ/Ω)·자료구조(스택·큐·힙·그래프)·재귀·증명기법(귀납·반증)·분할정복 |
| 의생명 AI | DNA/RNA·전사·발현·서열·k-mer·텐서·임베딩·그래프(노드·엣지)·attention |
| 의료진단 AI | 영상/신호·특징추출·민감도·특이도·ROC/AUC·과적합·교차검증 |

**용어 정의 형식 (각 1~3줄):**
- **용어** — 한 줄 정의. *쓰임:* 어느 수업 개념을 이해하는 데 쓰는지 연결.

예: **점근표기 O(n)** — 입력 크기 n에 대한 최악 실행시간의 상한. *쓰임:* 정렬·그래프 알고리즘의 복잡도 비교·증명에서 사용.

**규칙:**
- 정의만 나열하지 않는다. *쓰임* 으로 본문 개념과 연결한다(왜 알아야 하는지).
- 길어지는 설명은 본문 개념 섹션으로 미룬다(선행지식은 "최소 토대").
- 불확실한 정의는 단정하지 말고 강의자료 근거를 병기하거나 QA에 검증 요청.
- 선행지식 섹션도 목차 트리에 노드로 포함(id: `prereq` 하위).

## 산출물

1. `_workspace/{과목}_toc.md` — 탐색 트리용 계층 데이터(interactive-viz-engineer에게 전달)
2. 선행지식 섹션 HTML(대상 파일에 Edit 삽입)
3. 변경 요약 — 이동/병합/신설 항목, 바뀐 id 목록(있으면)

## 체크리스트

- [ ] 목차가 MECE하고 위상 정렬됨
- [ ] 선행지식 섹션이 본문 진입 전에 있고 각 용어에 *쓰임* 연결
- [ ] 기존 id 보존(또는 참조처까지 갱신)
- [ ] `_workspace/{과목}_toc.md`의 모든 id가 실제 섹션과 매칭
- [ ] 시험 포인트·예상문제 임의 삭제 없음
