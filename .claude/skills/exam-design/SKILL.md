# Exam Design — 실전 예상문제·모범답안

박스 안 3줄짜리 문제는 셀프테스트가 안 된다. 실제 기말 양식의 문제와 **채점 가능한 모범답안**을 만들어야 한다. 이 스킬은 과목별 출제 특성에 맞춘 문항·답안 양식을 담는다.

## 왜 이렇게 하나

- 문제가 실전 형식이 아니면 학습자는 "내가 정말 풀 수 있는지" 모른다. **출제 특성을 그대로 모사**해야 실전 점검이 된다.
- 정답만 있으면 틀린 이유를 못 배운다. **객관식은 오답 해설**, **서술형은 채점 포인트**가 학습의 핵심이다.

## 과목별 출제 양식

| 과목 | 교수 | 유형 | 양식 핵심 |
|---|---|---|---|
| 인공지능개론 | 최용석 | **서술형** | 토픽당 1문제. CSP Local Search 최우선. 모범답안 full-text |
| 컴퓨터 알고리즘 | 임을규 | **증명·복잡도** | 정리 증명·점근 분석. 증명 단계 번호·정당성 명시 |
| 의생명 AI | 노미나 | **서술형(입력→출력)** | 논문 3편 중심·GNN 식 확정. 주차당 2문제 |
| 의료진단 AI | 김정희 | **객관식 매칭형** | 개념↔정의 매칭·지표 선택. 보기+오답해설 |

## 문항 양식 표준

### 객관식 (의료진단 등)
```
[문제 N] {문두 — 실전 난이도}
  ① {보기}  ② {보기}  ③ {보기}  ④ {보기}  ⑤ {보기}

▶ 정답: ③
▶ 해설:
  - ③이 정답인 이유: {핵심 근거}
  - ① 틀린 이유 / ② 틀린 이유 / ④ ⑤ …  (각 오답이 왜 틀렸는지)
```
매칭형은 좌(개념)–우(정의) 대응표 + 정답 매핑 + 헷갈리는 쌍 구분 포인트.

### 서술형·증명 (AI개론·알고리즘·의생명)
```
[문제 N] {문두}

▶ 모범답안 (full-text):
  1. {정의/전제 — 영어(한글) 병기}
  2. {전개 — 단계별, 수식 포함}
  3. {결론}

▶ 채점 포인트: {이걸 쓰면 점수 — 필수 키워드·단계}
▶ 흔한 실수: {감점 포인트}
```
formatting 필수: 단계 번호·수식(KaTeX)·강조·필요 시 표. 줄글 3줄로 뭉개지 않는다.

## 작업 원칙

- **실전성.** 그 교수·그 과목 난이도·형식 그대로. 출제 가이드의 "시험 특성"을 반영.
- **모범답안은 채점 가능하게.** 채점 포인트(필수 키워드·단계)를 명시 → "이걸 쓰면 점수"가 보이게.
- **오답 해설 필수(객관식).** 각 오답이 왜 틀렸는지가 학습 가치.
- **개념 커버리지 매핑.** concept-explainer가 다룬 개념과 문항을 매핑 → 빈 영역·과잉 영역 점검.
- **영어(한글) 병기**([[explanation-style]]) — 문제·답안의 전문 용어도 적용.
- **셀프테스트 UI 유지.** 답 가림 → 펼침 토글(기존 패턴) 구조 보존, 새 문항엔 새 id.

## 안전 규칙

- 정답·증명 정확성 최우선. 틀린 정답·증명은 치명적. 불확실하면 근거 병기 + QA·사용자 검증. **추측 정답 금지.**
- 기존 토글 스크립트·id를 깨지 않게 통합.

## 산출물

- 대상 HTML 직접 Edit(예상문제 섹션) + 변경 요약(문항 수·유형 분포·대표 1문항 before/after)
- `_workspace/{과목}_exam.md` — 문항 메타(개념 커버리지 매핑)
