의료진단을 위한 AI (AAI0035)

이 과목은 결국 신경계 질환을 진단·재활하기 위해, 생체신호·의료영상을 측정(계측)하고 AI(ML/DL·CNN)로 해석하는 전체 파이프라인을 푼다.

먼저 잡을 큰 그림: 무엇을(신호·영상) → 어떻게 재는가(계측) → 무엇으로 해석하나(AI/신경망). 좌측 로드맵에서 노드를 클릭해 드릴다운하고, 추천 학습 경로 패널로 동선을 따라가세요.

⭐ 시험 특성(절대 우선): 객관식 70~80%, 매칭형 중심, 난이도 낮음. → 깊은 유도보다 정의·비교·매칭을 정확히 암기. 헷갈리는 쌍은 주황 테두리 매칭 대조표로 집중. 함정란의 "반대로 외우기 쉬운" 포인트를 특히 주의하세요.

B1. AI/ML/DL 기초 & 학습 패러다임

"데이터로 진단을 자동화하는 두뇌". 의료 AI의 사상부터 ML/DL 분류 체계까지. (C2가 관문, C2→C3→C4 순차)

C1 · 의료 AI 배경 / Human-centered AI / Ability-based Design ⭐中선행: 없음 ▶
🎯 직관"사람을 시스템에 맞추지 말고, 시스템을 사람의 능력에 맞춰라" — 이것이 Ability-based Design.
📖 정의Ability-based Design (Edwards 1995) — 일반 시스템은 대표 사용자를 상정해 사용자가 시스템에 적응(User→adaptation→System)한다. 반면 Ability-Based System(+AI)은 시스템이 사용자의 능력에 적응한다. 연구 3축: Custom Hardware + Signal Processing & AI + Human-centered Evaluation.
🌍 실세계 시나리오"손가락이 10개 없는 사용자가 QWERTY 키보드를 쓴다." 일반 설계는 사용자가 키보드에 맞춰 적응해야 하지만(User-adaptation-System), Ability-Based System은 AI가 그 사용자의 잔존 능력에 맞춰 입력 방식을 바꿔준다.
🔍 작동 원리연구 방향이 Diagnosis(진단) → Treatment(치료)로 이어짐. 치료는 다시 Therapeutic(비침습 신경조절) / Assistive(보조기술)로 나뉜다 — 신경계 질환 중심.
⚠️ 함정
반대로 외우기 쉬움: User-adaptation-System(사용자가 적응) ↔ Ability-Based System(시스템이 적응).
📝 시험 포인트 · 출제 가이드"Ability-based Design" 정의·키보드 예시 객관식. 진단 vs 치료(therapeutic=비침습 신경조절 / assistive=보조) 구분. (출처: Notion Day1 강조) · 🔗 응용 C20(Tremor 연구가 이 사상의 적용)
C2 · AI / ML / DL 계층 구조 ⭐⭐高매칭표관문 · 선행: 없음 ▶
🎯 직관러시아 인형처럼 AI ⊃ ML ⊃ DL. 바깥일수록 포괄적, 안쪽일수록 데이터로 자동 학습.
📖 정의
  • AI: 인간의 인지능력(학습·추론·판단·문제해결)을 기계가 수행. 규칙기반~자가학습 포괄.
  • ML: 데이터로 시스템이 스스로 규칙·패턴 학습(프로그래밍된 규칙 없이). 지도/비지도/강화로 분류.
  • DL: 다층 인공신경망 기반. 비정형 데이터(영상·음성·자연어)에 강력. 특징(feature) 자동 추출(End-to-End). CNN/RNN/Transformer.
🌍 실세계 시나리오응급실에서 — AI=환자 우선순위 분류·챗봇 / ML=혈액검사 수치로 질병 예측 / DL=흉부 X-ray를 CNN이 직접 보고 폐렴 진단. 셋 다 "AI"지만 안쪽으로 갈수록 데이터를 더 많이 먹고 사람 개입이 줄어든다.
🔍 작동 원리 (ML vs DL 핵심, Week1 p.26)ML = Feature engineering — Raw 데이터를 사람이 feature 설계 → ML모델 → 출력 (문제를 파트로 쪼개 답을 합침). DL = feature engineering 불필요 — Raw → DNN이 표현학습 → 출력.

✏️ 단계별 풀이 예제 (매칭형)
Q. 다음 특성을 AI/ML/DL 중 알맞은 것에 매칭하라: ① 흉부 X-ray를 raw로 넣어 폐렴 진단 ② 혈액검사 수치로 질병 예측 ③ black box라 설명이 어려움 ④ feature engineering이 필요
1단계 · ①
"raw 영상 직접 + 비정형" → 비정형에 강하고 feature 자동 = DL
2단계 · ②
"정형 수치 + 사람이 feature 설계" = ML
3단계 · ③
"설명가능성 낮음 = black box" = DL
4단계 · ④
"feature engineering 필요 = 사람이 설계" = ML (DL은 불필요)
답: ①DL ②ML ③DL ④ML
⚠️ 함정
DL은 ML의 하위 개념(AI⊃ML⊃DL). DL이 항상 우월한 게 아니다 — 데이터·연산을 많이 먹고, 과적합·설명력 낮음.
📝 시험 포인트⭐ 특성↔AI/ML/DL 매칭. 포함관계 AI⊃ML⊃DL. ML=feature engineering / DL=불필요. · 🔗 응용 C3, C16~C19(신경망=DL), 대조 C5
C3 · 학습 패러다임 — 지도 / 비지도 / 강화학습 ⭐⭐高매칭표선행: C2 ▶
🎯 직관정답 있음(지도) / 구조 발견(비지도) / 보상 최적화(강화).
📖 정의·메커니즘
  • 지도학습: 입력 X + 정답 y(label) 쌍 → 손실(Loss) 최소화로 파라미터 조정. 분류(SVM·Logistic·RF·k-NN·NN), 회귀(Linear·Ridge/Lasso·XGBoost).
  • 비지도학습: 입력만, 정답 없음 → 거리·유사도·분포로 내재 구조 발견. 클러스터링(K-means·DBSCAN·Hierarchical), 차원축소(PCA·t-SNE·Autoencoder).
  • 강화학습(RL): Agent가 Environment와 상호작용 → Reward로 최적 policy 학습. 기본 Q-learning·SARSA, 심화 DQN·PPO·A3C.
🌍 실세계 시나리오지도=X-ray로 폐렴 판단(정답표 있음) / 비지도=정상 생체신호만 모아두고 거기서 벗어나는 비정상 탐지(정답 없이 구조로) / 강화=재활 로봇이 시행착오로 보상을 쌓으며 동작 학습.

Q. 단점을 학습유형에 매칭: (가) 라벨링 비용이 크다 (나) 결과 평가가 어렵다 (다) 느리고 실험환경·윤리가 어렵다
1단계 · (가)
"라벨=정답이 필요" → 정답을 다 달아야 하니 = 지도학습
2단계 · (나)
"정답이 없어 잘했는지 모름" = 비지도학습

B2. 생체신호 (Biosignals)

"몸이 내는 전기·소리·진동을 읽는다". ECG·EEG·EMG가 핵심 삼총사. (C6 관문, 이후 C7·C8·C9·C10 병렬. C7·C8 매칭 1순위)

C6 · 생체신호 처리 개요 ⭐中관문 · 선행: 없음 ▶
🎯 직관몸이 내는 전기·소리·진동 신호를 다듬어 진단에 쓸 정보로 바꾸는 파이프라인.
📖 정의신호처리 = 소리·이미지·생체신호를 분석·변형·합성해 유용 정보 추출·품질 향상. 생체신호처리 = 인체 생리신호를 분석해 임상 진단·모니터링·치료에 활용. 대표 신호: ECG·EEG·EMG, 그 외 EOG·ENG·PCG·VAG.
🌍 실세계 시나리오ECG 원시 파형은 근육 잡음·기준선 흔들림이 섞여 있다. 필터링→정규화→R-피크 탐지→SVM/DL 분류를 거쳐야 "부정맥"이라는 진단 정보가 나온다.
🔍 작동 원리 (파이프라인, Week2 p.27)데이터수집 → 전처리(필터링·정규화·기준선보정) → 세분화(Segmentation) → 노이즈제거(Artifact Removal: ICA, PCA) → 피크탐지(예: ECG R-피크) → 분류·패턴인식(SVM·DL) → 모델링·예측 → 시각화.

📝 시험 포인트처리 파이프라인 순서. Artifact Removal = ICA/PCA. 라이브러리(SciPy=scipy.signal 필터, Scikit-learn=SVM/PCA). · 🔗 응용 C7·C8·C9·C10, C13(계측 시스템)

🎯 직관심장의 전기 신호를 피부에서 읽어 부정맥·심근경색을 잡는다.
📖 정의심장의 전기적 활동을 피부 표면에서 측정·기록. 리듬·속도·전도이상·허혈·심근경색 정보.

🌍 실세계 시나리오응급실에 흉통 환자가 오면 ECG에서 ST분절 상승을 보고 즉시 STEMI(심근경색)를 진단해 골든타임 내 시술로 보낸다.

Q. P파, QRS, T파를 각각 '심방 탈분극 / 심실 탈분극 / 심실 재분극'에 매칭하라.
1단계 · 순서

2단계 · P

3단계 · QRS

4단계 · T
마지막 = 심실의 재분극(회복).
답: P=심방 탈분극, QRS=심실 탈분극, T=심실 재분극
🔍 임상 활용 (Week2 p.10)①부정맥 ②심근경색(MI): ST상승=STEMI / 비ST상승=NSTEMI ③전도장애(방실차단·각차단 LBBB/RBBB) ④전해질이상(고칼륨→T파 고출현, 저칼슘→QT연장) ⑤심장비대.
⚠️ 함정
QRS는 "심실 탈분극"이지 재분극 아님. 재분극=T파. 심방 재분극은 QRS에 묻혀 안 보임.

2단계 · ①

3단계 · ②

4단계 · ③

⚠️ 함정
α=눈 감고 안정(각성 상태이되 휴식), β=집중. 둘을 헷갈리기 쉽다. P300은 "음"이 아니라 양(positive) 전위.

B3. 의료영상 모달리티

"몸 속을 들여다보는 11가지 카메라". 원리·방사선 유무·장단점 매칭이 시험 최다 출제. (독립 도입, 출력은 C19 CNN으로 연결)

C11 · 의료영상 모달리티 (11종) ⭐⭐⭐高 출제 최다매칭표 ×5독립 · 선행: 없음 ▶
🎯 직관몸 속을 보는 11가지 "카메라". 각각 원리·방사선 유무·주 대상·장단점이 다르고, 이 매칭이 시험에서 가장 많이 나온다.
📖 모달리티별 핵심
  • X-ray: 고에너지 전자파, 조직 밀도차 흡수(뼈=흰색). 골절·흉부·치과. 장점=빠름·저렴·골격 탁월 / 한계=연조직 해상도 낮음·방사선·2D.
  • CT: 여러 방향 X-ray를 컴퓨터 재구성 → 단면·3D. 뇌출혈·흉복부. 장점=고해상도·3D·빠름 / 한계=방사선·조영제·고가.
  • MRI: 강한 자기장+RF, 수소 원자핵(프로톤) 신호. 연조직 정밀(뇌·척수·관절). 확장 fMRI/DWI/DTI. 장점=방사선 없음·연조직 탁월 / 한계=고가·소음·금속 제한.
  • Ultrasound: 고주파 음파 반사. 실시간·태아·혈류(Doppler). 장점=방사선 없음·실시간·휴대·저렴 / 한계=공기·뼈 통과 어려움·숙련 의존.
  • PET: 방사성 동위원소(FDG=포도당 유사) → 대사 활발한 부위 집중 → 양전자→감마선. 대사·기능 영상. 장점=조기탐지·전이 탁월 / 한계=고가·방사성·시간 김.
  • SPECT: 감마선 방출체(Tc-99m, I-123) → 단일 광자 → 감마카메라 회전. 심근관류·뇌혈류. 장점=보편·저렴 / 한계=해상도 낮음(PET보다).
  • Mammography: 유방 전용 저용량 X-ray. 미세석회화 조기발견. 한계=젊은 여성 치밀유방 민감도 낮음.
  • Fluoroscopy: X-ray 실시간 연속 조사(C-arm). 조영제로 소화기·혈관. 한계=방사선 노출시간 증가.
  • DSA: 조영 전·후 영상을 디지털 감산 → 혈관만 선명. 뇌동맥류·AVM. 고급 fluoroscopy.
  • OCT: 근적외선 빛의 간섭 → μm 단위 초고해상도·비침습. 안과(망막)·혈관내시경. 한계=깊은 조직 어려움.
  • Dermatoscopy: 피부 병변 고배율(10~20배)+편광. ABCD Rule, AI 멜라노사이트 분류. 피부암 감별.

B4. 계측 시스템 (Instrumentation)

"신호를 어떻게 측정/제어하는가, 하드웨어 관점". 3블록 + 5대 분류. (C6에서 갈라짐, C9의 EMG-car가 C13의 실례)

C13 · 계측 시스템 3블록 모델 ⭐⭐⭐高선행: C6 ▶
🎯 직관몸의 신호를 측정/제어하는 3블록 파이프라인: 변환 → 전자 인터페이스 → 연산.
📖 정의 (3 Primary Subsystems)(1) Transducer → (2) Electronics Interface → (3) Computation Unit. (Tissue/생리신호와 Energy source 포함)
🌍 실세계 시나리오EMG Controlled Car — 근육 신호(Muscle)를 EMG 센서(Transducer)가 전압으로 바꾸고, 증폭·필터·정류(Electronics Interface)를 거쳐 ADC로 디지털화, 컴퓨터/MCU(Computation Unit)가 처리해 모터를 제어한다.
🔍 각 블록 세부
  • Transducer (Sensor + Actuator): 센서=생리신호 감지, 액추에이터=자극/제어. 대부분 voltage 신호 생성.
  • Electronics Interface (Pre-amplifier → Signal Conditioning → Buffer): A/D 변환 가능한 신호로. SNR·대역폭 보존.
    • Pre-amplifier: 저잡음·differential. 시스템 잡음 특성은 주로 pre-amp가 지배.
    • Signal Conditioning: 전압변환·증폭·필터링·정류·변조.
    • Buffer: 긴 케이블 구동. front-end와 back-end가 1~2m 이상 떨어질 때만 필요.
  • Computation Unit: UI·전체 제어·저장·신호처리·안전. (Data converter·DSP·Storage·UI·Comms·Control)
🔍 작동 원리 STEP-BY-STEP: 신호가 3블록을 통과하는 경로
1 · [직관]
몸의 신호는 약하고 잡음투성이 → 단계적으로 다듬어 컴퓨터가 읽게 한다.
2 · [셋업]
입력 = 생리신호(bioelectric 등). 목표 = 디지털 데이터/제어 명령.
3 · [전개 1] Transducer
생리신호를 전압으로 변환(센서) — 신호의 modality·source·dynamic range를 맞춤.
4 · [전개 2] Pre-amplifier
저잡음으로 증폭(여기서 시스템 잡음이 결정됨).
5 · [전개 3] Signal Conditioning
필터·정류·변조로 A/D 가능하게 다듬음.
6 · [전개 4] Buffer
(원거리면) Buffer가 케이블 구동. → ADC.
7 · [결론] Computation Unit
디지털 신호를 처리·저장·표시·제어.
8 · [시험 변형]
"잡음을 지배하는 블록은?"→Pre-amp. "Buffer가 필요한 조건?"→front/back-end 1~2m↑.
⚠️ 함정
잡음은 pre-amp가 지배(필터가 아님). Buffer는 항상 필요한 게 아니라 원거리(1~2m↑)에서만.
📝 시험 포인트⭐ 3블록 이름·순서, 각 기능. Pre-amp=잡음 지배. Buffer=원거리에서만. · 🔗 선행 C6, 예시 C9(EMG car), 응용 C14·C15
C14 · 측정·계측의 정의 ⭐中선행: C13 ▶
🎯 직관"측정 못하면 제어 못한다" — 재는 것이 모든 의료기기의 출발점.
📖 정의Measurement system = 물리·전기량(질량·온도·압력·전압) 측정, 사건/위치 탐지, 객체 식별·계수(적혈구). Instrumentation = 측정기기로 프로세스를 monitor·control 하는 기술. Biomedical Instrumentation = 몸으로부터/몸으로 데이터를 측정·기록·전송하는 기기 분야.
🔍 요구사항정확도·해상도·무게·부피·비용·환경.
📝 시험 포인트측정 시스템 요구사항. "측정 못하면 제어 못한다" 명제. Measurement vs Instrumentation vs Biomedical Instrumentation 구분. · 🔗 선행 C13, 응용 C15

B5. 신경망 & CNN

"AI가 실제로 학습하는 메커니즘". 뉴런부터 CNN까지 순차 누적. (C2에서 출발, C16→C17→C18→C19 엄격 순차. C19는 C11 영상을 입력으로 결합)

C16 · 인공뉴런 / 활성화함수 / 퍼셉트론 ⭐⭐高매칭표선행: C2 ▶
🎯 직관뉴런 = "입력에 가중치 곱해 더하고(가중합) + 비선형 함수로 굽히기".
🧮 수식 (뉴런, Week3 p.30)
\[ z = w_1 x_1 + w_2 x_2 + w_3 x_3 + b = \sum_i w_i x_i + b \]
입력 가중합 + 편향(bias). 이후 활성화함수 통과.
📖 활성화함수 (Week3 p.31)층 출력에 적용해 비선형성 부여. 없으면 NN이 선형회귀로 축소된다(아무리 층을 쌓아도).
📊 매칭 핵심 · 활성화함수 3종 (식·치역)
함수수식치역
Sigmoid\( f(x)=\dfrac{1}{1+e^{-x}} \)(0, 1)
TanH\( \tanh(x)=\dfrac{2}{1+e^{-2x}}-1 \)(−1, 1)
ReLU\( f(x)=\max(0,x) \)[0, ∞)
🔍 퍼셉트론 (Week3 p.32)뉴런 + 계단형 활성화(\( \sum wx+b\ge 0 \)이면 1, 아니면 0). 가중치 작은 변화가 큰 출력 변화 유발 → 다른(부드러운) 활성화함수 사용. 단일 퍼셉트론은 XOR 표현 불가. MLP = 여러 퍼셉트론·완전연결층 + 다양한 활성화함수.
🌍 실세계 시나리오폐렴 판별 뉴런 — X-ray 특징들(x)에 가중치(w)를 곱해 합치고, Sigmoid로 0~1 확률로 굽혀 "폐렴일 확률 0.87"을 낸다.
⚠️ 함정
활성화함수가 없으면 선형회귀(비선형성이 핵심 이유). 단일 퍼셉트론은 XOR 불가 → MLP가 필요했던 역사적 이유. Sigmoid 치역=(0,1), TanH=(−1,1) 헷갈리지 말 것.
📝 시험 포인트⭐ 활성화함수 목적(비선형성, 없으면 선형회귀). Sigmoid/TanH/ReLU 식·치역 매칭. 단일 퍼셉트론 XOR 불가. · 🔗 선행 C2, 응용 C17
C17 · 신경망 구조 / 파라미터 수 계산 / 행렬표현 ⭐⭐高계산형선행: C16 ▶
🎯 직관각 층의 파라미터 수 = (출력노드 수) × (입력노드 수 + 1) — +1은 bias.
🧮 수식 (파라미터 수, Week3 p.34)
\[ \text{한 층 파라미터} = n_{out} \times (n_{in} + 1) \]
전체 = 각 층 합.
✏️ 단계별 풀이 예제 (파라미터 수 계산 — 시험 계산형!)
Q. 입력 6노드 → 1st hidden 4 → 2nd hidden 3 → output 1. 총 파라미터 수는?
1단계 · 1st hidden
출력 4 × (입력 6 + 1) = 4 × 7 = 28
2단계 · 2nd hidden
출력 3 × (입력 4 + 1) = 3 × 5 = 15
3단계 · output
출력 1 × (입력 3 + 1) = 1 × 4 = 4
4단계 · 합산
28 + 15 + 4 = 47
답: 47 (bias를 빼먹으면 안 됨!)
🧮 행렬표현 (Week3 p.35)
\[ x=[x_1,x_2,x_3]=a^{(1)},\quad z^{(2)}=xW^{(1)},\quad a^{(2)}=f(z^{(2)}),\ \dots,\ y=\text{softmax}(z^{(4)}) \]
softmax는 마지막 층 선호(값을 확률로 해석 가능).
⚠️ 함정
bias의 +1을 빼먹는 것이 가장 흔한 계산 실수. softmax는 은닉층이 아니라 출력층.
📝 시험 포인트⭐ 파라미터 수 = Σ (출력 × (입력+1)). softmax=출력층(확률 해석). · 🔗 선행 C16, 응용 C18
C18 · Forward / Backward Propagation / 학습 절차 ⭐⭐高선행: C17 ▶
🎯 직관앞으로(forward) 가서 예측하고, 틀린 만큼 뒤로(backward) 돌아가며 가중치를 고친다.
📖 정의
  • Forward propagation: 입력 X → 은닉층 \( a^{(2)},a^{(3)},a^{(4)} \) 순차 계산 → 출력 \( \hat{y} \).
  • Backward propagation: 손실 \( J(y,\hat{y}) \) → 출력층 가중치부터 역순으로 gradient \( \frac{\partial J}{\partial W_4}, \dots, \frac{\partial J}{\partial W_1} \) 계산(체인룰, 역순 전파).
🔍 작동 원리 STEP-BY-STEP: 학습 5단계 (Week3 p.43 TRAINING STEPS)
1 · [Feed network]
입력·batch·epoch 설정, 가중치 초기화
2 · [Make prediction]
forward로 출력 \( \hat{y} \) 생성
3 · [Compare with ground truth]
손실함수 J 계산 (\( y \) vs \( \hat{y} \))
4 · [Calculate gradients & Update]
\( \partial J/\partial w_i \) 계산, backpropagation으로 가중치 갱신
5 · [Iterate & monitor]
반복하며 수렴·과적합 감시
📊 그래프 인사이트 (Week3 p.44) — 진단형 출제!
Loss vs epoch (학습률 진단): very high LR → 발산(상승) / low LR → 너무 느림 / high LR → 정체(플래토) / good LR → 부드럽게 수렴.
Accuracy vs epoch (과적합 진단): training↑인데 validation이 크게 뒤처지면 strong overfitting, 약간 뒤처지면 little overfitting.
✏️ 단계별 풀이 예제 (Loss 곡선 진단)
Q. Loss가 epoch 따라 위로 치솟으면 무엇이 문제?
1단계
Loss는 줄어야 정상인데 올라감 = 발산.
2단계
발산의 원인 = 학습률이 너무 큼(very high LR).
답: 학습률을 낮춰야 함.
⚠️ 함정
forward=입력→출력, backward=손실→gradient(반대 방향). 손실 J는 4단계가 아니라 3단계에서 계산(예측 직후 비교).
📝 시험 포인트⭐ forward/backward 방향. 학습 5단계 순서. Loss 곡선→학습률, Accuracy 곡선→과적합 진단. · 🔗 선행 C17, 응용 C19
C19 · CNN (합성곱 신경망) / 이미지 분석 ⭐⭐高종착 · 선행: C18, C11 ▶
🎯 직관이미지는 평평하게 펴면 공간 정보가 깨진다 → 필터를 슬라이드하며 공간 구조를 보존하는 게 CNN.
📖 왜 CNN? (Week3 p.45–48)이미지를 1D로 펴서 FC(완전연결)에 넣으면 FC는 공간 구조를 무시한다(픽셀을 permute해도 가중치만 permute되고 결과 동일). → 공간 보존 위해 CNN.
🔍 합성곱 메커니즘 (Week3 p.49–53)입력=color channels(RGB, zero-padding) → kernels(필터)와 합성곱 + bias → output channel. 여러 kernel → 여러 output channel. CNN은 뉴런을 3차원(width·height·depth)으로 배열(일반 NN은 1D).
🌍 실세계 시나리오흉부 X-ray 폐렴 진단 — 첫 층 필터가 가장자리·선을 잡고, 깊은 층이 폐의 음영 패턴을 잡아 "폐렴" 분류. C11의 영상이 그대로 CNN 입력이 된다.
📊 표 인사이트 — 필터 예시 (Week3 p.54)
Vertical Line Detector / Horizontal Line Detector / Corner Detector (각 3×3). Filter(kernel) = 입력과 합성곱되는 가중치 집합.
🔍 과적합 감소 (Week3 p.55)데이터 증강(data augmentation) — rotation·flip·crop·stretch·translate·resize·contrast·blur·RGB↔BGR·color inversion.
⚠️ 함정
FC가 공간 구조를 무시하는 것이 CNN의 존재 이유(반대로 외우지 말 것). 데이터 증강은 과적합을 줄인다. CNN 뉴런은 3D 배열.
