🎓 선행지식 — 본문 전 최소 토대

본문 개념(신호·영상·계측·신경망)을 읽기 전에 알아야 할 용어. 각 정의 1~3줄 + 쓰임으로 본문과 연결한다. 진단 성능지표(민감도·특이도·ROC/AUC)는 객관식 매칭형에 직결되니 아래 임계값 슬라이더로 직접 조작해 체득하라.

P1. 영상 / 신호선행▶
📖 정의신호(signal) = 시간에 따라 변하는 1차원 측정값(ECG·EEG·EMG 등 생체전기). 영상(image) = 공간에 배열된 2D/3D 픽셀값(X-ray·CT·MRI 등). 둘 다 진단의 원자료(raw data)다.
🔗 쓰임B2(생체신호)는 1D 신호, B3(의료영상)은 2D/3D 영상을 다룬다. CNN(C19)은 영상의 공간 구조를, 신호처리(C6)는 시간 구조를 보존한다.
P2. 특징추출 (Feature Extraction)선행▶
📖 정의raw 데이터에서 분류·예측에 쓸모 있는 수치(특징, feature)를 뽑는 과정. 사람이 설계하면 feature engineering, 모델이 스스로 뽑으면 표현학습(representation learning)이다.
🔗 쓰임ML과 DL을 가르는 핵심 축(C2): ML은 사람이 feature를 설계, DL은 자동 추출. CNN(C19)이 영상의 특징을 자동으로 잡는 것이 대표 예다.
P3. 민감도 · 특이도 (Sensitivity · Specificity)⭐핵심▶
📖 정의진단 모델을 환자/정상으로 판정할 때의 두 적중률.
민감도(Sensitivity, 재현율) = 실제 환자를 환자로 맞힌 비율 = TP / (TP + FN).
특이도(Specificity) = 실제 정상을 정상으로 맞힌 비율 = TN / (TN + FP).
📊 혼동행렬(Confusion Matrix)TP(진양성)·FP(위양성)·TN(진음성)·FN(위음성) 네 칸. 민감도는 놓친 환자(FN)를, 특이도는 오진한 정상(FP)을 줄일수록 올라간다.
🔗 쓰임의료 진단의 표준 평가지표. 임계값을 낮추면 민감도↑·특이도↓로 맞교환(trade-off)이 생긴다 — 아래 P4 슬라이더로 직접 확인하라.
P4. ROC · AUC + 임계값 슬라이더⭐핵심·인터랙티브▶
📖 정의ROC 곡선 = 임계값을 0→1로 훑으며 (1−특이도, 민감도)를 찍은 곡선. x축 = 위양성률(FPR = 1−특이도), y축 = 민감도(TPR). AUC = 곡선 아래 면적(0.5=무작위, 1.0=완벽). 임계값과 무관하게 모델 전체 분별력을 한 숫자로 요약한다.
🎛️ 직접 조작 — 임계값(threshold)을 바꿔 보라

임계값↑ → 환자로 판정하기 까다로워짐 → 특이도↑·민감도↓. 임계값↓ → 민감도↑·특이도↓.

슬라이더를 움직이면 혼동행렬·민감도·특이도와 ROC 위의 현재 점이 즉시 갱신된다.

⚠️ 함정
ROC의 x축은 특이도가 아니라 위양성률(1−특이도)다. 임계값을 바꿔도 AUC(곡선 자체)는 바뀌지 않는다 — AUC는 모델 고유 값, 임계값은 곡선 위의 한 점을 고를 뿐이다.
🔗 쓰임분류(C4)·신경망(C18) 성능 평가의 표준. 매칭형으로 "민감도↔TP/(TP+FN)", "임계값↓→민감도↑·특이도↓" 관계를 묻기 좋다.
P5. 과적합 (Overfitting)선행▶
📖 정의모델이 학습 데이터에 지나치게 맞춰져 새 데이터(검증·시험)에서 성능이 떨어지는 현상. 학습 정확도는 높은데 검증 정확도가 크게 뒤처지면 과적합이다.
🔗 쓰임학습곡선 진단(C18)의 핵심 — train↑·validation 정체면 strong overfitting. CNN(C19)은 데이터 증강으로 과적합을 줄인다. DL이 항상 우월하지 않은 이유(C2)이기도 하다.
P6. 교차검증 (Cross-Validation)선행▶
📖 정의데이터를 여러 묶음(fold)으로 나눠, 번갈아 학습/검증에 써서 성능을 평균내는 방법. k-fold가 대표적이다. 단일 분할의 운(運)을 줄여 일반화 성능을 더 믿을 만하게 추정한다.
🔗 쓰임과적합(P5) 여부를 안정적으로 판단하고 모델·하이퍼파라미터를 고를 때 쓴다. 의료처럼 데이터가 적을 때 특히 중요하다.

B1. AI/ML/DL 기초 & 학습 패러다임

C1. 의료 AI 배경 / Human-centered AI / Ability-based Design⭐中선행: 없음▶
🎯 직관"사람을 시스템에 맞추지 말고, 시스템을 사람의 능력에 맞춰라" — 이것이 Ability-based Design.
📖 정의Ability-based Design (Edwards 1995) — 일반 시스템은 대표 사용자를 상정해 사용자가 시스템에 적응(User→adaptation→System)한다. 반면 Ability-Based System(+AI)은 시스템이 사용자의 능력에 적응한다. 연구 3축: Custom Hardware + Signal Processing & AI + Human-centered Evaluation.
🌍 실세계 시나리오
"손가락이 10개 없는 사용자가 QWERTY 키보드를 쓴다." 일반 설계는 사용자가 키보드에 맞춰 적응해야 하지만(User-adaptation-System), Ability-Based System은 AI가 그 사용자의 잔존 능력에 맞춰 입력 방식을 바꿔준다.
🔍 작동 원리연구 방향이 Diagnosis(진단) → Treatment(치료)로 이어짐. 치료는 다시 Therapeutic(비침습 신경조절) / Assistive(보조기술)로 나뉜다 — 신경계 질환 중심.
⚠️ 함정
User-adaptation-System(사용자가 적응) ↔ Ability-Based System(시스템이 적응)을 반대로 외우기 쉽다.
📝 시험 포인트 (출제 가이드 · Notion Day1)"Ability-based Design" 정의·키보드 예시 객관식. 진단 vs 치료(therapeutic=비침습 신경조절 / assistive=보조) 구분.
🔗 연결: 응용 C20(Tremor 연구가 이 사상의 적용)
C2. AI / ML / DL 계층 구조⭐⭐高선행: 없음 · 관문▶
🎯 직관러시아 인형처럼 AI ⊃ ML ⊃ DL. 바깥일수록 포괄적, 안쪽일수록 데이터로 자동 학습.
📖 정의AI: 인간의 인지능력(학습·추론·판단·문제해결)을 기계가 수행. 규칙기반~자가학습 포괄.
ML: 데이터로 시스템이 스스로 규칙·패턴 학습(프로그래밍된 규칙 없이). 지도/비지도/강화로 분류.
DL: 다층 인공신경망 기반. 비정형 데이터(영상·음성·자연어)에 강력. 특징(feature) 자동 추출(End-to-End). CNN/RNN/Transformer.
🌍 실세계 시나리오
응급실에서 — AI=환자 우선순위 분류·챗봇 / ML=혈액검사 수치로 질병 예측 / DL=흉부 X-ray를 CNN이 직접 보고 폐렴 진단. 셋 다 "AI"지만 안쪽으로 갈수록 데이터를 더 많이 먹고 사람 개입이 줄어든다.
🔍 작동 원리 (ML vs DL 핵심, Week1 p.26)ML = Feature engineering — Raw 데이터를 사람이 feature 설계 → ML모델 → 출력. DL = feature engineering 불필요 — Raw → DNN이 표현학습 → 출력.

C3. 학습 패러다임 — 지도 / 비지도 / 강화학습⭐⭐高선행: C2▶
🎯 직관정답 있음(지도) / 구조 발견(비지도) / 보상 최적화(강화).
📖 정의·메커니즘지도학습: 입력 X + 정답 y(label) 쌍 → 손실(Loss) 최소화로 파라미터 조정. 분류(SVM·Logistic·RF·k-NN·NN), 회귀(Linear·Ridge/Lasso·XGBoost).

강화학습(RL): Agent가 Environment와 상호작용 → Reward로 최적 policy 학습. 기본 Q-learning·SARSA, 심화 DQN·PPO·A3C.

⭐ 매칭 대조표: ECG 파형 P / QRS / T
파형의미설명
P파심방 탈분극심방 수축 위한 전기자극
QRS 복합파심실 탈분극심실 수축 위한 전기자극
T파심실 재분극심실이 휴식상태로 복귀
U파(드묾)심실 마지막 재분극명확하지 않음
정량: 1회 심박동 0.8~1초, 분당 60~80회. PR interval 0.12–0.20s, QRS 0.06–0.10s, QTc ≤0.44s.
✏️ 단계별 풀이 예제 (파형 매칭) "P파·QRS·T파를 '심방 탈분극 / 심실 탈분극 / 심실 재분극'에 매칭하라."
1단계
순서: 심장은 심방 먼저 → 심실 나중, 각각 탈분극(수축) → 재분극(회복).
2단계
P = 가장 먼저 = 심방의 탈분극.
3단계
QRS = 크고 가운데 = 심실의 탈분극(수축).
4단계 → 답
T = 마지막 = 심실의 재분극(회복). ∴ P=심방탈분극, QRS=심실탈분극, T=심실재분극
🔍 임상 활용 (Week2 p.10)①부정맥 ②심근경색(MI): ST상승=STEMI / 비ST상승=NSTEMI ③전도장애(방실차단·각차단 LBBB/RBBB) ④전해질이상(고칼륨→T파 고출현, 저칼슘→QT연장) ⑤심장비대.
⚠️ 함정
QRS는 "심실 탈분극"이지 재분극 아님. 재분극=T파. 심방 재분극은 QRS에 묻혀 안 보임.
📝 시험 포인트⭐ P/QRS/T ↔ 심방탈분극/심실탈분극/심실재분극 매칭. STEMI vs NSTEMI. 전해질이상 ECG 변화.
🔗 연결: 선행 C6, 대조 C8(EEG)

⭐⭐ 매칭 대조표: EEG 5대역 δθαβγ ↔ 상태
파형주파수상태
Delta (δ)0.5–4 Hz깊은 수면
Theta (θ)4–8 Hz졸림·깊은 명상
Alpha (α)8–13 Hz안정·깨어있음(눈 감음)

외우기: 주파수 낮을수록 잠(δ수면), 높을수록 각성·인지(γ). 알파벳 순 δ→θ→α→β→γ = 주파수 오름차순.
✏️ 단계별 풀이 예제 (대역↔상태 매칭) "다음 상태를 EEG 대역에 매칭: ① 깊은 수면 ② 눈 감고 안정 ③ 집중해 문제 풀이"
1단계
규칙: 주파수 낮음=수면, 높음=각성·집중.
2단계
① 깊은 수면 = 가장 낮은 = Delta(δ, 0.5–4Hz)
3단계
② 눈 감고 안정 = Alpha(α, 8–13Hz)
4단계 → 답
③ 집중·문제해결 = Beta(β, 13–30Hz). ∴ ①δ ②α ③β
🔍 P300 (ERP)자극 후 ~300ms에 나타나는 양(positive) 전위. 활용: 인지기능 평가(알츠하이머·ADHD), 조현병·우울증(진폭 감소), BCI P300 Speller.
⚠️ 함정
α=눈 감고 안정(각성 상태이되 휴식), β=집중. 둘을 헷갈리기 쉽다. P300은 "음"이 아니라 양(positive) 전위.

B3. 의료영상 모달리티

C11. 의료영상 모달리티 (11종)⭐⭐⭐高선행: 없음 · 출제 최다▶
🎯 직관몸 속을 보는 11가지 "카메라". 각각 원리·방사선 유무·주 대상·장단점이 다르고, 이 매칭이 시험에서 가장 많이 나온다.
📖 모달리티별 핵심
  • X-ray: 고에너지 전자파, 조직 밀도차 흡수(뼈=흰색). 골절·흉부·치과. 빠름·저렴·골격 탁월 / 연조직 해상도 낮음·방사선·2D.
  • CT: 여러 방향 X-ray를 컴퓨터 재구성 → 단면·3D. 뇌출혈·흉복부. 고해상도·3D·빠름 / 방사선·조영제·고가.
  • MRI: 강한 자기장+RF, 수소 원자핵(프로톤) 신호. 연조직 정밀(뇌·척수·관절). 확장 fMRI/DWI/DTI. 방사선 없음 / 고가·소음·금속 제한.
  • Ultrasound: 고주파 음파 반사. 실시간·태아·혈류(Doppler). 방사선 없음·실시간·휴대·저렴 / 공기·뼈 통과 어려움·숙련 의존.
  • PET: 방사성 동위원소(FDG=포도당 유사) → 대사 활발한 부위 집중 → 양전자→감마선. 대사·기능 영상. 조기탐지·전이 탁월 / 고가·방사성·시간 김.
  • SPECT: 감마선 방출체(Tc-99m, I-123) → 단일 광자 → 감마카메라 회전. 심근관류·뇌혈류. 보편·저렴 / 해상도 낮음(PET보다).
  • Mammography: 유방 전용 저용량 X-ray. 미세석회화 조기발견. 젊은 여성 치밀유방 민감도 낮음.
  • Fluoroscopy: X-ray 실시간 연속 조사(C-arm). 조영제로 소화기·혈관. 방사선 노출시간 증가.
  • DSA: 조영 전·후 영상을 디지털 감산 → 혈관만 선명. 뇌동맥류·AVM. 고급 fluoroscopy.
  • OCT: 근적외선 빛의 간섭 → μm 단위 초고해상도·비침습. 안과(망막)·혈관내시경. 깊은 조직 어려움.
  • Dermatoscopy: 피부 병변 고배율(10~20배)+편광. ABCD Rule, AI 멜라노사이트 분류. 피부암 감별.

B4. 계측 시스템 (Instrumentation)

C13. 계측 시스템 3블록 모델⭐⭐⭐高선행: C6▶
🎯 직관몸의 신호를 측정/제어하는 3블록 파이프라인: 변환 → 전자 인터페이스 → 연산.
📖 정의 (3 Primary Subsystems)(1) Transducer → (2) Electronics Interface → (3) Computation Unit. (Tissue/생리신호와 Energy source 포함)
🌍 실세계 시나리오
EMG Controlled Car — 근육 신호(Muscle)를 EMG 센서(Transducer)가 전압으로 바꾸고, 증폭·필터·정류(Electronics Interface)를 거쳐 ADC로 디지털화, 컴퓨터/MCU(Computation Unit)가 처리해 모터를 제어한다.
🔍 블록별 세부
  • Transducer (Sensor + Actuator): 센서=생리신호 감지, 액추에이터=자극/제어. 대부분 voltage 신호 생성.
  • Electronics Interface (Pre-amplifier → Signal Conditioning → Buffer): A/D 변환 가능한 신호로. SNR·대역폭 보존. — Pre-amp: 저잡음·differential, 시스템 잡음 지배 / Signal Conditioning: 전압변환·증폭·필터링·정류·변조 / Buffer: 긴 케이블 구동, front-end·back-end 1~2m 이상 떨어질 때만 필요.
  • Computation Unit: UI·전체 제어·저장·신호처리·안전. (Data converter·DSP·Storage·UI·Comms·Control)
🔍 STEP-BY-STEP: 신호가 3블록을 통과하는 경로
1. 직관
몸의 신호는 약하고 잡음투성이 → 단계적으로 다듬어 컴퓨터가 읽게 한다.
2. 셋업
입력 = 생리신호(bioelectric 등). 목표 = 디지털 데이터/제어 명령.
3. 전개 1
Transducer가 생리신호를 전압으로 변환(센서) — modality·source·dynamic range를 맞춤.
4. 전개 2
Pre-amplifier가 저잡음으로 증폭(여기서 시스템 잡음이 결정됨).
5. 전개 3
Signal Conditioning이 필터·정류·변조로 A/D 가능하게 다듬음.
6. 전개 4
(원거리면) Buffer가 케이블 구동. → ADC.
7. 결론
Computation Unit이 디지털 신호를 처리·저장·표시·제어.
8. 시험 변형
"잡음을 지배하는 블록은?"→Pre-amp. "Buffer가 필요한 조건?"→front/back-end 1~2m↑.
⚠️ 함정
잡음은 pre-amp가 지배(필터가 아님). Buffer는 항상 필요한 게 아니라 원거리(1~2m↑)에서만.
📝 시험 포인트⭐ 3블록 이름·순서, 각 기능. Pre-amp=잡음 지배. Buffer=원거리에서만.

C16. 인공뉴런 / 활성화함수 / 퍼셉트론⭐⭐高선행: C2▶
🎯 직관뉴런 = "입력에 가중치 곱해 더하고(가중합) + 비선형 함수로 굽히기".

📖 활성화함수 (Week3 p.31)층 출력에 적용해 비선형성 부여. 없으면 NN이 선형회귀로 축소된다(아무리 층을 쌓아도).

함수수식치역
Sigmoid\( f(x)=\dfrac{1}{1+e^{-x}} \)(0, 1)
TanH\( \tanh(x)=\dfrac{2}{1+e^{-2x}}-1 \)(−1, 1)

🔍 퍼셉트론 (Week3 p.32)뉴런 + 계단형 활성화(\( \sum wx+b\ge 0 \)이면 1, 아니면 0). 가중치 작은 변화가 큰 출력 변화 유발 → 다른(부드러운) 활성화함수 사용. 단일 퍼셉트론은 XOR 표현 불가. MLP = 여러 퍼셉트론·완전연결층 + 다양한 활성화함수.
🌍 실세계 시나리오
폐렴 판별 뉴런 — X-ray 특징들(x)에 가중치(w)를 곱해 합치고, Sigmoid로 0~1 확률로 굽혀 "폐렴일 확률 0.87"을 낸다.
⚠️ 함정
활성화함수가 없으면 선형회귀(비선형성이 핵심 이유). 단일 퍼셉트론은 XOR 불가 → MLP가 필요했던 역사적 이유. Sigmoid 치역=(0,1), TanH=(−1,1) 헷갈리지 말 것.
📝 시험 포인트⭐ 활성화함수 목적(비선형성, 없으면 선형회귀). Sigmoid/TanH/ReLU 식·치역 매칭. 단일 퍼셉트론 XOR 불가.

✏️ 단계별 풀이 예제 (파라미터 수 계산 — 시험 계산형!) 입력 6노드 → 1st hidden 4 → 2nd hidden 3 → output 1. 총 파라미터 수는?
1단계
1st hidden: 출력 4 × (입력 6 + 1) = 4 × 7 = 28
2단계
2nd hidden: 출력 3 × (입력 4 + 1) = 3 × 5 = 15
3단계
output: 출력 1 × (입력 3 + 1) = 1 × 4 = 4
4단계 → 답
합산: 28 + 15 + 4 = 47 (bias를 빼먹으면 안 됨!)
🧮 행렬표현 (Week3 p.35)
\[ x=[x_1,x_2,x_3]=a^{(1)},\quad z^{(2)}=xW^{(1)},\quad a^{(2)}=f(z^{(2)}),\ \dots\ y=\text{softmax}(z^{(4)}) \]
softmax는 마지막 층 선호(값을 확률로 해석 가능).
⚠️ 함정
bias의 +1을 빼먹는 것이 가장 흔한 계산 실수. softmax는 은닉층이 아니라 출력층.
📝 시험 포인트⭐ 파라미터 수 = Σ (출력 × (입력+1)). softmax=출력층(확률 해석).
🔗 연결: 선행 C16, 응용 C18
C18. Forward / Backward Propagation / 학습 절차⭐⭐高선행: C17▶
🎯 직관앞으로(forward) 가서 예측하고, 틀린 만큼 뒤로(backward) 돌아가며 가중치를 고친다.
📖 정의Forward propagation: 입력 X → 은닉층 \( a^{(2)},a^{(3)},a^{(4)} \) 순차 계산 → 출력 \( \hat{y} \).
Backward propagation: 손실 \( J(y,\hat{y}) \) → 출력층 가중치부터 역순으로 gradient \( \frac{\partial J}{\partial W_4}, \dots, \frac{\partial J}{\partial W_1} \) 계산(체인룰, 역순 전파).
🔍 STEP-BY-STEP: 학습 5단계 (Week3 p.43 TRAINING STEPS)
1. Feed network
입력·batch·epoch 설정, 가중치 초기화
2. Make prediction
forward로 출력 \( \hat{y} \) 생성
3. Compare with ground truth
손실함수 J 계산 (\( y \) vs \( \hat{y} \))
4. Calculate gradients & Update
\( \partial J/\partial w_i \) 계산, backpropagation으로 가중치 갱신
5. Iterate & monitor
반복하며 수렴·과적합 감시
📊 그래프 인사이트 (Week3 p.44) — 진단형 출제!
Loss vs epoch (학습률 진단): very high LR → 발산(상승) / low LR → 너무 느림 / high LR → 정체(플래토) / good LR → 부드럽게 수렴.
Accuracy vs epoch (과적합 진단): training↑인데 validation이 크게 뒤처지면 strong overfitting, 약간 뒤처지면 little overfitting.
✏️ 단계별 풀이 예제 (Loss 곡선 진단) "Loss가 epoch 따라 위로 치솟으면 무엇이 문제?"
1단계
Loss는 줄어야 정상인데 올라감 = 발산.
2단계 → 답
발산의 원인 = 학습률이 너무 큼(very high LR) → 학습률을 낮춰야 함.
⚠️ 함정
forward=입력→출력, backward=손실→gradient(반대 방향). 손실 J는 4단계가 아니라 3단계에서 계산(예측 직후 비교).
📝 시험 포인트⭐ forward/backward 방향. 학습 5단계 순서. Loss 곡선→학습률, Accuracy 곡선→과적합 진단.

C19. CNN (합성곱 신경망) / 이미지 분석⭐⭐高선행: C18, C11▶
🎯 직관이미지는 평평하게 펴면 공간 정보가 깨진다 → 필터를 슬라이드하며 공간 구조를 보존하는 게 CNN.
📖 왜 CNN? (Week3 p.45–48)이미지를 1D로 펴서 FC(완전연결)에 넣으면 FC는 공간 구조를 무시한다(픽셀을 permute해도 가중치만 permute되고 결과 동일). → 공간 보존 위해 CNN.
🔍 합성곱 메커니즘 (Week3 p.49–53)입력=color channels(RGB, zero-padding) → kernels(필터)와 합성곱 + bias → output channel. 여러 kernel → 여러 output channel. CNN은 뉴런을 3차원(width·height·depth)으로 배열(일반 NN은 1D).
🌍 실세계 시나리오
흉부 X-ray 폐렴 진단 — 첫 층 필터가 가장자리·선을 잡고, 깊은 층이 폐의 음영 패턴을 잡아 "폐렴" 분류. C11의 영상이 그대로 CNN 입력이 된다.
📊 표 인사이트 — 필터 예시 (Week3 p.54)
Vertical Line Detector / Horizontal Line Detector / Corner Detector (각 3×3). Filter(kernel) = 입력과 합성곱되는 가중치 집합.
🔍 과적합 감소 (Week3 p.55)데이터 증강(data augmentation) — rotation·flip·crop·stretch·translate·resize·contrast·blur·RGB↔BGR·color inversion.
⚠️ 함정
FC가 공간 구조를 무시하는 것이 CNN의 존재 이유(반대로 외우지 말 것). 데이터 증강은 과적합을 줄인다. CNN 뉴런은 3D 배열.
📝 시험 포인트⭐ "FC는 공간 무시 → CNN 필요". Filter=합성곱 가중치 집합. CNN=3D(W·H·depth). 데이터 증강=과적합 감소.
